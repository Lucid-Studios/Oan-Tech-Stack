# SANCTUARYID GOA DEFAULT REFUSAL LAW

## Purpose

This note defines refusal as the first governance competence of
`SanctuaryID.GoA` in `V1.2.1`.

It exists so governance root standing does not overread readiness as reach.

## Governing Compression

> GoA working is not CME spawn working.
> refusal is the first lawful governance competence whenever requested
> authority outruns the lower floor that already stands.

## Default Refusal Rule

Refusal is the default whenever a requested act assumes:

- predicate promotion that is still withheld
- engram minting
- active governing `CME` presence
- CradleTek-dependent life
- service authorization not explicitly receipted

Refusal must be visible and receipted.

## Refusal Classes

The first refusal classes in this phase are:

- predicate-maturity refusal
- minting refusal
- governing-presence refusal
- CradleTek-dependency refusal
- unauthorized-service refusal

These classes keep overreach legible without counterfeiting permission.

## Explicit Non-Grants

This note does not permit:

- convenience-based acceptance
- temporary allow
- borrowed maturity
- silent fallback into later-life assumptions
- unreceipted governance reach

## Working Summary

`SanctuaryID.GoA` now defaults to refusal whenever requested authority exceeds
the lower floor that already stands.
That keeps governance root standing lawful before any spawn, body, or service
authorization is discussable.
