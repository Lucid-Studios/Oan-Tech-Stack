# SLI_GOVERNED_RETURN_RECEIPT_FAMILY_LAW

## Purpose

This note defines the first governed-return receipt families carried into
`V1.2.1`.

It exists so reviewed descent is not treated as one generic callback, one
generic approval, or one generic notice surface.

Its job in this line is to fix:

- the first governed-return receipt families
- the standing each receipt carries
- acknowledgement burden
- local transformability limits

## Governing Compression

`GovernedReturn` is not the fact that something came back.

It is the lawful downward transport of reviewed standing.

The first line-local read is:

> a governed-return receipt is a typed judgment-bearing surface that re-enters
> bounded operation with explicit standing, obligations, and limits.

## Install-First Boundary

`V1.2.1` is not yet claiming live reviewed descent handling.

What is admitted now is:

- line-local governed-return doctrine
- a first passive receipt-family contract surface

What remains withheld is:

- live receipt landing
- mission-time acknowledgement enforcement
- local packaging into later operator surfaces

## First Receipt Family

The first line-local receipt family now reads:

- `PermissionReceipt`
- `RestrictionReceipt`
- `CorrectionReceipt`
- `RedirectionReceipt`
- `CollapseReceipt`
- `DeferReceipt`
- `RefusalNoticeReceipt`
- `QuarantineNoticeReceipt`
- `TransformedInstructionReceipt`

These are carried line-locally in:

- `src/SLI/SLI.Runtime/GovernedReturnReceiptContracts.cs`

## Required Standing

Every governed-return receipt in this line should preserve at least:

- `receipt_id`
- `return_family`
- `source_review_jurisdiction`
- `source_state_lineage`
- `trace_id`
- `admissibility_posture`
- `authority_ceiling`
- `burden_of_return`
- `required_acknowledgement`
- `expiry_or_represent_window`
- `local_transformability_rule`

## Acknowledgement Burden

The first bounded acknowledgement family now reads:

- `AcknowledgeOnly`
- `AcknowledgeAndComplianceRecord`
- `AcknowledgeAndWitnessLog`
- `AcknowledgeAndRepresentPacket`

## Local Transformability Rule

The first bounded transformability family now reads:

- `VerbatimOnly`
- `RenderOnly`
- `BoundedLocalPackaging`

No receipt in this line defaults to free reinterpretation.

## Hard Prohibitions

1. No governed return may be treated as generic permission.
2. No local layer may widen reviewed standing by convenience.
3. No transformed instruction may be restored to its superseded form locally.
4. No receipt may become ambient standing by omission of burden or expiry.

## Implementation Consequences

This note now governs the first passive governed-return receipt family in
`V1.2.1`.

The carried batch gives the line:

- typed receipt families
- typed acknowledgement burdens
- typed transformability rules
- typed receipt packets with explicit reviewed standing fields

It does not yet claim:

- live receipt landing
- automatic descent into active runtime lanes
- governed-return compliance automation
