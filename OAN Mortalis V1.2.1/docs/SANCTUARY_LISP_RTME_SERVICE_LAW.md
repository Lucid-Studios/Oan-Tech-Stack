# SANCTUARY_LISP_RTME_SERVICE_LAW

## Purpose

This note defines the first encrypted Lisp `RTME` as a `Sanctuary`-hosted
service layer in `V1.2.1`.

It exists to keep five things distinct:

- template `RTME` office
- issued `SanctuaryID.RTME` service line
- `CradleTek` runtime binding
- `SoulFrame` bridge-and-seal work
- `CME` formation

## Governing Compression

> The first Lisp `RTME` is a `Sanctuary` service layer.
> `Sanctuary.RTME` names the service-form template.
> `SanctuaryID.RTME` names the issued, certified, continuity-bearing hosted
> service line.
> `CradleTek` does not own that service; it later binds through it.

## Office

`Sanctuary.RTME` is:

- the service-form template office
- the doctrinal or archetypal runtime medium before issuance

`SanctuaryID.RTME` is:

- the issued hosted Lisp service line
- the actual service surface used for forward symbolic calls when needed
- a continuity-bearing surface subject to issuance, certification, and
  cryptographic lineage

## Service Rule

The first `RTME` should be read as:

- hosted by `Sanctuary`
- available only after lawful `Sanctuary.GEL` bootstrap
- callable as a service layer during later runtime work

It should not be read as:

- a `CradleTek`-owned runtime organ
- a free-floating worker
- a replacement for governance or issuance law

## Bind Rule

Later runtime families reach the hosted `RTME` lawfully:

- through `SanctuaryID.GoA`
- downward into `CradleTekID.GoA`
- through the pre-established Steward Nexus control manifold
- through the `SoulFrame cGoA` bridge required for cryptic sealing

This means `CradleTek` uses the hosted service during `SoulFrame` build-out.
It does not own or originate it.

## ID Rule

`SanctuaryID.RTME` should exist only when:

- issuance is complete
- certification is complete
- cryptographic lineage has been attached

No runtime surface should use `*ID` decoratively.

## Non-Collapse Rules

- `Sanctuary.RTME` is not `SanctuaryID.RTME`
- `SanctuaryID.RTME` is not `CradleTekID.GoA`
- `SanctuaryID.RTME` is not a `CME`
- `CradleTek` may bind through the hosted service, but it does not originate
  the service office
- no `CME` should be described as if it were the first `RTME`
