# INSTALL_AGREEMENT_ACTION_SURFACE_LAW

## Purpose

This note defines the first lawful install-assent action surface in
`OAN Mortalis V1.2.1`.

It exists so localized install choice is not flattened into raw text,
ambient contract presence, or silent identity minting.

## Governing Compression

> assent capture is not identity minting, but no install identity may exist
> without assented predicate formation.

> localized agreement predicates must be formed from the authorized choice
> matrix and lawful agreement surface, not inferred from raw text or default
> convenience.

> mutation candidate is not mutation warrant.

## Cryptic Bundle Seat

The install-agreement action surface is seated as a cryptic bundle extension
surface beneath `SLI.Lisp`.

In this phase it is:

- `installed-disabled`
- not part of the canonical floor set
- not a live hosted service
- not a runtime-authorization surface

## Action Sequence

The first lawful action sequence now reads:

`LocalizedInstallChoiceMatrix -> AgreementPredicateBundle -> InstallIdentitySetCandidate -> CoreCmeUsePostureRecord`

## Input Surface

The surface consumes one `LocalizedInstallChoiceMatrix`.

That matrix binds:

- selected language/localization dataset
- W5H and legal footing section
- agreement template/version lineage
- licensing-agent identity
- user identity
- per-lane assent state and witness refs

## Output Offices

The action surface may emit only:

- `AgreementPredicateBundle`
- `InstallIdentitySetCandidate`
- `CoreCmeUsePostureRecord`

No direct persistence, service activation, Atlas mutation, or runtime
authorization is granted by emission.

This footing may later braid into site-bound pre-Cradle authorization only
through separate site, operator, disclosure, and content-authority surfaces.

## Identity Boundary

`InstallIdentitySetCandidate` may appear only after full assent across every
required agreement lane.

`Acknowledged`, `Presented`, `Refused`, or `Withheld` standing is insufficient
for identity formation.

## Explicit Non-Grants

This note does not authorize:

- live hosted service binding
- install identity minting from acknowledgement only
- Atlas delta candidacy
- Atlas mutation
- `Sanctuary.MoS` handoff
- site-bound pre-Cradle authorization by install footing alone
- paid-access branch logic
- runtime activation

## Working Summary

`InstallAgreementActionSurface` is now the first lawful assent surface for
localized install footing.

It is seated in the cryptic hosted layer as an extension surface, but it
remains installed-disabled and non-executive in this phase.
