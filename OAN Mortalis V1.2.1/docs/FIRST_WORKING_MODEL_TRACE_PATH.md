# FIRST_WORKING_MODEL_TRACE_PATH

## Purpose

This note defines the first witnessed, non-executive path from later
operator-ingress admissibility toward the named
`SLI.Engine -> SLI.Lisp -> SanctuaryID.RTME` seam.

It exists to prove passage order and receipt order without granting runtime
behavior.

## Governing Compression

The working sentences are:

> trace is not execution.

> admissibility may approach the seam; only receipt proves what happened
> there.

This note therefore describes one lawful path and one refusal branch only.
It does not generalize into live runtime motion.

## Governing Nouns

The path is restricted to already admitted nouns:

- `SLI.Runtime.RecompositionCandidate`
- `SLI.Runtime.RecompositionCandidateEvaluationDecision`
- membrane re-entry burden
- `SLI.Engine.PredicateLandingRequest`
- `SLI.Engine.CrypticFloorEvaluation`
- `SLI.Lisp.HostedCrypticLispBundleReceipt`
- seam refusal receipt as defined conceptually in
  `SEAM_REFUSAL_AND_RETURN_LAW.md`

The current line still does not materialize a separate
`San.Common.MembraneDecisionResult`.
Membrane decision standing therefore remains visible through
`San.Common.MembraneDecision` on `PredicateLandingRequest`.

## Witnessed Path

The first witnessed path is:

`RecompositionCandidate -> RecompositionCandidateEvaluationDecision -> membrane re-entry -> PredicateLandingRequest -> CrypticFloorEvaluation -> HostedCrypticLispBundleReceipt`

### Step 1. Candidate Standing

The path begins only from a `RecompositionCandidate` that remains:

- `Disposition == CandidateOnly`
- provenance-bearing
- trace-bearing
- office-legible

### Step 2. Evaluation Standing

The path continues only when the paired
`RecompositionCandidateEvaluationDecision` remains:

- `Outcome == RetainCandidate`
- `EligibleForLaterOperatorRealization == true`
- free of failed `Constitutional` burden
- free of failed `Continuity` burden

At this point the path is still only admissibility-bearing.
It has not crossed into realization.

### Step 3. Membrane Re-Entry Burden

The path then preserves explicit membrane re-entry burden.

No candidate may bypass that burden by being well formed.
No evaluation result may silently upgrade itself into seam authority.

### Step 4. Predicate Landing Request

The first bounded seam carrier is then formed as:

- `SLI.Engine.PredicateLandingRequest`

That carrier must still preserve:

- `SymbolicEnvelope`
- `MembraneDecision`
- `SanctuaryGelHandle`
- `IssuedRtmeHandle`
- `RouteHandle`
- `RouteKind`

### Step 5. Engine Witness

`SLI.Engine.CrypticFloorEvaluation` then proves what the engine says about the
carrier.

The first successful witnessed path requires:

- `PredicateLandingReady == true`
- `Disposition == Ready`
- `OutcomeCode == "predicate-landing-ready"`

The governance trace may presently read:

- `predicate-landing-surface-ready-via-direct-transit`
- or `predicate-landing-surface-ready-via-bounded-ec-transit`

### Step 6. Hosted Bundle Receipt

Only after the earlier path remains intact may the current hosted-side receipt
prove resident bundle posture:

- `SLI.Lisp.HostedCrypticLispBundleReceipt`

The first witnessed path treats the receipt as proof of:

- hosted bundle identity
- hosted-by-issued-runtime posture
- hosted-execution-only posture
- canonical floor-set readiness

The path stops here.
Receipt is the witness outcome.
It is not live execution.

## Refusal Branch

The first refusal branch is:

`RecompositionCandidate -> RecompositionCandidateEvaluationDecision -> membrane re-entry -> PredicateLandingRequest -> CrypticFloorEvaluation -> seam refusal receipt`

The simplest current refusal example is missing issued runtime identity.

In that branch:

- `PredicateLandingRequest` exists
- `IssuedRtmeHandle` is missing or blank
- `CrypticFloorEvaluation` returns `Disposition == Withhold`
- `OutcomeCode == "issued-rtme-service-required"`
- `GovernanceTrace == "issued-sanctuary-rtme-service-required-before-engine-landing"`

At that point the path may not continue toward hosted receipt.
It must instead return as a seam refusal receipt witnessed under
`SEAM_REFUSAL_AND_RETURN_LAW.md`.

## Explicit Non-Powers

This trace path does not:

- prove live binding
- prove live `RTME` wake
- prove live `EC`
- prove operator realization
- authorize action because the path is coherent

The path proves only that the line can witness lawful approach and lawful
refusal without silent drop.
