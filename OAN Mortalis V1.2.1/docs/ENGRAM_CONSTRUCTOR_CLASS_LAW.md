# ENGRAM_CONSTRUCTOR_CLASS_LAW

## Purpose

This note defines the constructor-class ladder for `Sanctuary.GEL`.

It exists to give `GEL` its first lawful interior build order without letting
constructor growth collapse into semantic layer claims, categorical engram
claims, or runtime consequence.

## Governing Compression

> constructor class answers how far the engram has been built; categorical
> class answers what kind of semantic office it holds.
>
> semantic formation layer is not constructor class.

## Constructor Ladder

The only constructor-class ladder admitted in this phase is:

`Root -> Basic -> Intermediate -> Advanced -> Master`

No new constructor classes are admitted in this phase.

### `Root`

`Root` hosts:

- intake anchors
- ancestry-bearing references
- symbol-preserved roots
- primitive relations only

`Root` does not host full propositional or procedural standing.

### `Basic`

`Basic` hosts:

- first propositional engrams
- minimal procedural refusal/check engrams

`Basic` does not host continuity-bearing synthesis.

### `Intermediate`

`Intermediate` hosts:

- linked propositions
- bounded procedures
- first lawful correspondence handling

`Intermediate` does not host capstone integration.

### `Advanced`

`Advanced` hosts:

- situational and correspondence-aware assemblies
- governed method bundles

`Advanced` does not replace `Master`.

### `Master`

`Master` is the capstone continuity-bearing synthesis.

It may integrate already-admitted propositions, already-bounded procedures,
continuity trace, and universe-bearing synthesis without becoming runtime
authority by default.

## Constructor And Categorical Distinction

Constructor class is distinct from:

- semantic formation layer
- categorical engram class

Constructor class answers build depth.

Categorical class answers semantic office.

In this phase, categorical discussion may include:

- `propositional`
- `procedural`

Those categorical classes do not replace the constructor ladder.

The symbolic transport-form cluster now governs `Root`, `Basic`, and
`Intermediate` transport construction for canonical root carriers and governed
super/sub formation.

That transport cluster does not widen `Advanced` or `Master`, and it does not
replace the constructor ladder with transport grammar.

## Class Ceilings And Floors

The constructor-class floors and ceilings in this phase are:

- `Root` may carry ancestry, preserved symbol standing, and primitive relation
  hints only
- `Basic` is the first constructor class that may host full propositional
  standing or minimal procedural refusal/check law
- `Intermediate` is the first constructor class that may host bounded
  correspondence handling across multiple engrams
- `Advanced` is the first constructor class that may host situational
  assemblies and governed method bundles
- `Master` is the only admitted capstone constructor class

## Canonical Base Fields

Every engram in this phase should carry at minimum:

- `engram_id`
- `constructor_class`
- `categorical_class`
- `universe_binding`
- `ancestry_refs[]`
- `witness_refs[]`
- `admissibility_state`

## Explicit Non-Grants

This note does not grant:

- new constructor classes
- semantic identity by class label alone
- proposition standing at `Root`
- procedural execution authority
- predicate promotion
- engram minting
- runtime activation

## Working Summary

`Sanctuary.GEL` now has an admitted constructor-class ladder.

That ladder fixes how far an engram has been built without confusing build
depth with semantic formation layer, categorical office, or runtime standing.
