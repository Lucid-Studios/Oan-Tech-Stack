# FIELD_QUERY_TENSION_AND_LAWFUL_RECOMPOSITION_LAW

## Purpose

This note mirrors lawful retrieval and recomposition into the `V1.2.1`
install-first line.

It exists so the sibling line does not become asymmetrical:

- strict on ingress
- loose on retrieval

Without this note, the new line could admit symbolic products lawfully and
then later recall and combine them carelessly.

## Governing Compression

Retrieval is a second membrane crossing.
Queried field products remain passported, modality-typed, and
authority-bounded.
No query may elevate them.
No recomposition may occur without lawful re-entry.

The strongest install-first sentence is:

> Query must not imply rewrite.
> Retrieved products remain what they were under passport law.
> Recomposition must preserve provenance explicitly.
> No recomposition candidate may bypass membrane re-entry.

## Install-First Boundary

What is admitted now is:

- line-local retrieval and recomposition doctrine
- a first passive query family under `src/SLI/SLI.Runtime/`
- a bounded query engine that inspects and summarizes rather than rewrites
- explicit recomposition candidates with provenance and membrane re-entry
  burden

What remains withheld is:

- live field mutation
- automatic recomposition landing
- operator realization that acts over recomposed candidates
- any claim that retrieval alone grants persistence, promotion, or continuity

## Bidirectional Membrane Read

The membrane should now be read in two crossings.

### First Crossing: Ingress

`raw -> adapter -> envelope -> membrane -> lanes`

### Second Crossing: Retrieval For Recomposition

`lanes -> read model -> query tension -> membrane re-entry -> recomposition candidate`

The law stays the same in both directions:

- no bypass
- no hidden authority widening
- no passport erasure
- no counterfeit promotion

## First Passive Runtime Family

The first passive retrieval family now carried in this line lives in:

- `src/SLI/SLI.Runtime/FieldQueryContracts.cs`
- `src/SLI/SLI.Runtime/FieldQueryPolicy.cs`
- `src/SLI/SLI.Runtime/QueryTensionSummary.cs`
- `src/SLI/SLI.Runtime/RecompositionCandidateContracts.cs`

This family now gives the line:

- `FieldQuery`
- `FieldQueryResult`
- `IFieldQueryEngine`
- `QueryTensionSummary`
- `RecompositionCandidate`

These remain:

- typed
- bounded
- passport-preserving
- non-sovereign

## Hard Rules

1. Query must not imply rewrite.
2. No retrieval may bypass the membrane.
3. No query may elevate the authority of retrieved data.
4. Recomposition must preserve provenance explicitly.
5. Recomposition must remain subordinate to preserved-field law.
6. No recomposition candidate may act, persist, or claim standing without
   lawful membrane re-entry.

## Runtime Consequence

The first passive `SLI.Runtime` query family should therefore be read as:

- lawful inspection of held field products
- explicit tension summary
- explicit candidate-only recomposition
- preserved passport truth
- preserved authority ceilings

It is not:

- a side door around witness
- a hidden normalization engine
- a silent memory healer
- an alternate promotion path

The next passive seam after this family is candidate evaluation, not operator
realization.

## Working Summary

`V1.2.1` now carries a first passive field-query and recomposition family
beneath preserved-field law.
Query remains non-sovereign.
Recomposition remains provenance-bearing and candidate-only.
Any later action still requires lawful membrane re-entry.
