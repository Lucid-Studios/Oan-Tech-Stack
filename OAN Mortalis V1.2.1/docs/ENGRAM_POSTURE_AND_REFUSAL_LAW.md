# ENGRAM_POSTURE_AND_REFUSAL_LAW

## Purpose

This note defines the only lawful non-runtime posture set for this phase of
`Sanctuary.GEL`.

It exists to keep semantic condition and governance condition from being read
as execution.

## Governing Compression

> posture is descriptive-governing, not executable.
>
> refusal remains explicit and witnessed.

## Posture Classes

The only admitted posture classes in this phase are:

- `admit`
- `provisional`
- `hold`
- `refuse`

## Posture Derivation

Posture derives from:

- semantic condition
- governance condition

Semantic condition includes:

- proposition state
- ambiguity
- absence
- malformation
- correspondence/contradiction state

Governance condition includes:

- applicable procedures
- hostile pressure
- continuity state
- refusal law

## Refusal Receipt Rule

Refusal must remain:

- explicit
- witnessed
- receipted

No silent refusal, silent downgrade, or invisible bypass is admitted.

## Canonical Fields

Every posture record should carry at minimum:

- `posture_id`
- `frame_ref`
- `posture_class`
- `semantic_condition_refs[]`
- `governance_condition_refs[]`
- `refusal_reason_refs[]`
- `witness_refs[]`
- `next_review_condition`

## Explicit Non-Grants

This note does not grant:

- execution authority
- runtime transition
- service authorization
- predicate promotion
- engram minting

## Working Summary

`Sanctuary.GEL` now fixes its first lawful posture set.

That posture set remains descriptive-governing, explicit, witnessed, and
non-runtime.
