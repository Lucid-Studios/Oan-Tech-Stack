# SYMBOLIC_TRANSPORT_LINEAGE_AND_CONTEXT_SPLIT_LAW

## Purpose

This note defines when symbolic transport lineage remains one object and when
later local-universe context may justify a split.

It exists to keep transport ancestry stable while refusing both automatic
surface collapse and premature contextual divergence.

## Governing Compression

> Separation must be earned by context, formation, and witnessed use.
>
> Transport readiness preserves root ancestry and formation law; it does not
> guarantee local semantic realization.

## Lineage Office

This office governs the transport-level relation between one canonical root
carrier, its governed formations, and later possible contextual differentiation.

Its work is to preserve:

- one transport lineage
- governed formation ancestry
- candidate universe plurality
- witnessed use before split

It is not allowed to:

- treat surface variation as automatic split
- treat shared ancestry as automatic local identity
- guarantee downstream semantic behavior from transport law alone

## Non-Split Rule

Transport preserves ancestry and formation discipline.
It does not guarantee downstream local semantic behavior.

One transport lineage should remain one transport object unless context,
formation, and witnessed use lawfully justify a later split discussion.

Surface variation alone is insufficient.
Shared ancestry alone is insufficient for local identity.

## Split Triggers

Local-universe context may later decide whether one lineage remains one
transport object or becomes a distinct contextual object.

A split must be earned by:

- context
- formation
- witnessed use

This phase admits only the following transport-level split states:

- `same-lineage`
- `context-pending`
- `split-eligible`
- `split-refused`

Canonical fields:

- `transport_object_id`
- `root_symbol_ref`
- `formation_refs[]`
- `universe_candidates[]`
- `lineage_state`
- `context_split_state`
- `use_witness_refs[]`
- `correspondence_refs[]`

## Refusal Rules

The following must force refusal, hold, or non-promotion:

- automatic split from surface variation
- automatic local identity from shared ancestry
- split claim without witnessed use
- split claim without formation pressure
- transport law treated as proof of local semantic realization

## Explicit Non-Grants

This note does not grant:

- final contextual semantic identity
- Atlas delta candidacy
- Atlas mutation
- predicate promotion
- engram minting
- operator realization
- runtime activation

## Working Summary

`SLI` now has a lineage-and-context split note for symbolic transport.

That note keeps transport lineage shared by default, makes split an earned
result rather than an automatic one, and refuses to promise local semantic
realization from transport law alone.
