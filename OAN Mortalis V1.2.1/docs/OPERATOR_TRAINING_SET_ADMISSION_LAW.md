# OPERATOR TRAINING SET ADMISSION LAW

## Purpose

This note defines operator training-set admission as a required standing
surface for pre-Cradle site authorization in `V1.2.1`.

It exists so operator presence is not mistaken for trained admissibility.

## Governing Compression

> operator training-set completion is required standing for pre-Cradle
> authorization, but it is not operator realization.

## Admission Office

The bounded passive operator training record now lives in:

- `OperatorTrainingAdmissionRecord`

That record binds:

- one operator
- one training set
- one admission standing
- one site-binding profile
- one witness chain

Operator training-set admission must remain tied to a concrete site profile.
Abstract operator presence is insufficient.

## Existing Seam Dependence

Use this note together with:

- `CRADLETEK_SITE_BINDING_PROFILE_LAW.md`
- `OPERATOR_INGRESS_PRECONDITIONS.md`
- `PRE_CRADLETEK_SITE_AUTHORIZATION_CANDIDATE_LAW.md`

Pre-Cradle operator training admission is earlier than later operator
realization ingress and does not replace that later seam.

## Explicit Non-Grants

This note does not grant:

- operator realization
- governing `CME` placement
- CradleTek authorization
- live runtime governance

## Working Summary

`V1.2.1` now requires operator training-set admission to be site-bound,
witnessed, and distinct from later operator realization.
