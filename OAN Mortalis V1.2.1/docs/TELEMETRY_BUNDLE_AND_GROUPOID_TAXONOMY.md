# TELEMETRY_BUNDLE_AND_GROUPOID_TAXONOMY

## Purpose

This note fixes the end-to-end telemetry taxonomy for the install-first
`OAN Mortalis V1.2.1` line.

It exists so the new production-form sibling can inherit one lawful telemetry
read before its `.audit` chamber matures into a thicker runtime-bearing body.

This note does not claim that `V1.2.1` already carries the same emitted
telemetry burden as `V1.1.1`.

It fixes the classes early so later surfaces line up with release state
instead of inventing a second vocabulary under pressure.

## Governing Compression

The governing correction is:

> `bundle` names packaging, not semantic carrier class.

That means:

- a `receipt` may be packaged inside a run bundle
- a `ledger` may be snapshotted into a state surface
- a `witness` may be emitted through a report readout
- a `candidate_packet` may be retained inside a daily bundle

The packaging does not redefine the semantic class.

The second governing correction is:

> every emitted telemetry surface should be readable through one shared
> identity tuple rather than through filename habit alone

## Semantic Carrier Classes

The active semantic carrier classes for this line are:

- `standing_surface`
- `appendix_packet`
- `receipt`
- `ledger`
- `witness`
- `envelope`
- `summary_digest`
- `candidate_packet`

These classes are fixed now even when some line-local emitted surfaces remain
undeclared or unavailable.

## Packaging Classes

The active packaging classes for this line are:

- `state_surface`
- `run_bundle`
- `daily_bundle`
- `report_readout`

## Telemetry Groupoids

The active telemetry surfaces for this line should now be read through:

- `domain`
  `Sanctuary -> CradleTek -> SoulFrame -> AgentiCore`
- `spline`
  `Install -> Build -> Run -> Rest -> Exit`
- `semanticClass`
  `standing_surface -> appendix_packet -> receipt -> ledger -> witness -> envelope -> summary_digest -> candidate_packet`
- `authorityClass`
  `transport -> evidence -> candidate -> admitted -> inherited`
- `continuityClass`
  `event -> session -> thread -> line -> sibling-line`
- `retentionClass`
  `rolling_state -> hourly_raw -> daily_compacted -> pinned_review -> durable`
- `packageClass`
  `state_surface -> run_bundle -> daily_bundle -> report_readout`

## Required Identity Tuple

Every emitted telemetry surface should now be readable as:

- `{ domain, spline, semanticClass, authorityClass, continuityClass, retentionClass, packageClass }`

Where the current line does not yet declare or emit a field, later readouts
must say:

- `undeclared`
  when the line has not named it yet
- `unavailable`
  when the line names it but current data is absent

## Current Install-First Read

The current install-first line already declares:

- a reserved `.audit/` chamber
- line-local readiness
- line-local doctrine braid
- line-local source families

The current line does not yet declare a thick active emitted telemetry body
comparable to `V1.1.1`.

That means this taxonomy is presently release-alignment law first and emitted
surface law second.

## Relation To Later Readouts

Later read-only report surfaces such as:

- `line-audit-report`
- `line-readiness-report`
- `line-braid-report`
- `line-trace-readout`

should read this taxonomy rather than invent a private one.

The first fixed read-only schema for that family now lives in:

- `LINE_AUDIT_REPORT_SCHEMA.md`

## Non-Goals

This note does not:

- claim that `V1.2.1` already has full active telemetry lanes
- promote install-first doctrine into runtime authority
- replace later line-local telemetry contracts when they arrive

## Working Summary

`V1.2.1` now shares the same end-to-end telemetry taxonomy as the active line,
while remaining truthful that most emitted telemetry in this sibling line is
still forming.
