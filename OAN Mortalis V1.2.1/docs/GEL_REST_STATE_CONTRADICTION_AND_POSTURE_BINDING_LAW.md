# GEL_REST_STATE_CONTRADICTION_AND_POSTURE_BINDING_LAW

## Purpose

This note defines how contradiction and posture attach to the resting object
inside `Sanctuary.GEL`.

It exists to keep contradiction and posture bound at rest without turning
either into action.

## Governing Compression

> contradiction at rest is a bound condition, not a transient parser event.
>
> posture at rest is descriptive-governing, not executable method.

## Contradiction Binding

Contradiction attaches to the resting object as:

- a bound resting condition
- a continuity-bearing condition
- a review-bearing condition

It does not become a runtime event by attachment alone.

## Posture Binding

Posture attaches to the resting object as:

- descriptive-governing state
- bounded rest-state posture
- review-bearing state

It does not become action by attachment alone.

## Coexistence Law

Multiple unresolved conditions may coexist lawfully at rest without collapse.

That includes:

- contradiction plus ambiguity
- contradiction plus provisional posture
- contradiction plus unresolved correspondence

## Canonical Fields

Every contradiction/posture binding should carry at minimum:

- `binding_id`
- `held_object_ref`
- `contradiction_refs[]`
- `posture_ref`
- `coexistence_state`
- `review_requirements[]`
- `witness_refs[]`

## Explicit Non-Grants

This note does not grant:

- action
- execution
- promotion
- runtime consequence
- `Sanctuary.MoS` standing

## Working Summary

`Sanctuary.GEL` now fixes contradiction and posture as bound resting
conditions.

Those conditions may coexist lawfully without collapsing into action,
promotion, or runtime implication.
