# MOS_STORAGE_SEAT_REFUSAL_AND_BOUNDARY_LAW

## Purpose

This note fixes the refusal edges and anti-collapse boundaries for the
`Sanctuary.MoS / Sanctuary.cMoS` storage-seat corridor.

It exists so storage-seat standing is not overstated into pre-Cradle
authorization, governing placement, or independent cryptic sovereignty.

## Governing Compression

> storage-seat standing is not pre-Cradle authorization.
>
> pre-Cradle authorization is not governing `CME` placement.
>
> `cOE` and `cSelfGEL` may not be read as a second sovereign origin.

## Refusal Rule

The following interpretations must be refused in this phase:

- resting `GEL` persistence as automatic `MoS` standing
- `cGEL` as legal/governance seat
- `MoS/cMoS` as governing `CME` placement
- storage-seat standing as CradleTek authorization
- `cOE/cSelfGEL` as independent sovereignty

The bounded passive refusal family for this corridor now lives in:

- `MosStorageSeatRefusalReason`

## Boundary Rule

When any memory-seat claim is:

- absent
- implied rather than receipted
- borrowed from nearby law
- overstated into runtime or governance life

the lawful response is:

- preserve the prior lower standing
- preserve `CmePlacementWithheld`
- refuse any authorization overread
