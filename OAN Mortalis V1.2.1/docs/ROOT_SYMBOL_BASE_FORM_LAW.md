# ROOT_SYMBOL_BASE_FORM_LAW

## Purpose

This note defines the canonical root carrier at transport base form for
Atlas-derived symbolic material.

It exists to ensure that root-bearing transport identity remains singular,
adjunct-free, and shared across the admitted universe ring before any later
contextual split is considered.

## Governing Compression

> Root symbols are canonical, singular, and adjunct-free at base form.
>
> shared root ancestry does not automatically imply identical local semantic
> realization.

## Base-Form Office

This office governs the transport base form of a root symbol.

Its work is to preserve:

- one canonical transport carrier
- one root ancestry reference
- one singular base form
- witnessed surface variation without automatic split

It is not allowed to:

- admit multiple competing canonical carriers for one admitted transport root
- add super or sub extension at base form
- add operator ornament at base form
- treat contextual split as part of base form

## Shared-Carrier Rule

This phase admits one shared canonical transport carrier across the admitted
universe ring.

That shared carrier preserves transport lineage.
It does not guarantee that later local-universe realization will remain
identical.

Surface variation does not automatically imply separate engram identity.
Shared root ancestry does not automatically imply identical local semantic
realization.

## Constructor Eligibility

Only `Root` constructor class is admitted at transport base form.

`Root` base form must remain:

- singular
- adjunct-free
- ancestry-bearing
- witnessable across admitted universes

Canonical fields:

- `root_symbol_id`
- `canonical_symbol`
- `constructor_class = Root`
- `root_ancestry_ref`
- `base_form_state = adjunct-free`
- `surface_variants[]`
- `witness_refs[]`

## Refusal Rules

The following must force refusal, hold, or non-promotion:

- second competing canonical root carrier
- super or sub extension attached at `Root`
- operator ornament attached at `Root`
- contextual split claimed at base form
- surface variation treated as automatic identity split

## Explicit Non-Grants

This note does not grant:

- contextual semantic identity by ancestry alone
- `Basic` or `Intermediate` formation by implication
- Atlas delta candidacy
- Atlas mutation
- predicate promotion
- engram minting
- runtime activation

## Working Summary

`SLI` now has a root-symbol base-form note for transport law.

That note fixes one shared canonical carrier at `Root`, keeps base form
adjunct-free, and refuses both premature contextual split and false promises
of identical local semantic realization.
