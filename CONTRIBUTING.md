# Contributing

## Scope

This repository accepts public documentation and release-boundary work only. It
is not the place to submit implementation-bearing pull requests, private build
materials, runtime payloads, or reconstruction guides.

## Suitable Contributions

Suitable public contributions include:

- clearer public explanations;
- safer non-claim language;
- documentation hygiene;
- issue and pull request template improvements;
- responsible disclosure routing;
- public release-boundary clarification;
- non-executable examples that do not reveal private implementation.

## Out Of Scope

Do not submit:

- source code intended to reconstruct the private build;
- build, install, test, release, or deployment automation;
- datasets, fixtures, generated audit outputs, or runtime payloads;
- private corpus references or raw logs;
- local absolute paths, credentials, tokens, secrets, or machine-specific
  configuration;
- implementation-bearing architecture contracts, schemas, symbolic maps, or
  operating sequences.

## Pull Requests

Each pull request should state:

- what public wording changed;
- why it improves clarity, safety, reviewability, or accountability;
- what it does not claim;
- whether any implementation-bearing detail was intentionally withheld.

## Issues

Use public issues for documentation clarity, public boundary questions, and
repository hygiene. Do not use public issues to request private build
instructions, internal file maps, local runtime details, or executable
reproduction steps.

Security concerns and sensitive disclosure should be reported through the
repository security policy rather than a public issue.

## Public Boundary

Contribution does not grant identity, custody, governance authority, legal
authority, release authority, production readiness, or access to private build
materials.

Maintainers may close, redirect, or redact public issues and pull requests that
risk exposing private implementation, sensitive data, or controlled research
material.

## Maintainer Contact

For repository administration, contribution routing, or maintainer-side
coordination that should not start as a public issue, use the published
maintainer contact path for Lucid Studios.
