# Contributing

## Scope

This repository accepts public documentation and disclosure-boundary work only.
It is not the active engineering workspace and does not accept implementation
pull requests.

## Appropriate Contributions

Appropriate contributions include:

- correcting public overview wording;
- improving safety, governance, or accountability explanations;
- clarifying non-claims;
- improving responsible disclosure guidance;
- proposing non-executable public examples.

## Out of Scope

Do not submit:

- source code or build scripts;
- test fixtures or datasets;
- deployment, release, or automation instructions;
- internal architecture specifications;
- private corpus, identity, custody, operator, or runtime material;
- local machine paths, logs, secrets, tokens, or credentials.

## Pull Requests

Each pull request should state:

- what public wording changed;
- why it improves safety or clarity;
- what it does not claim;
- whether any private or implementation detail was intentionally withheld.

## Issues

Use public issues only for documentation clarity, public safety wording, and
release-boundary questions. Security concerns and sensitive disclosure should be
reported privately through [SECURITY.md](SECURITY.md).
