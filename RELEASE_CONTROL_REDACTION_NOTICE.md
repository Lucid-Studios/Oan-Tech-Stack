# Release Control Redaction Notice

Date: 2026-04-26
Updated: 2026-05-06
Scope: public repository and public issue tracker surface
Status: descriptive public release posture

## Summary

The public repository has been reduced from an implementation-bearing workspace
to a documentation-only technology description surface.

Earlier public materials included buildable source, internal documentation,
automation scripts, fixture data, symbolic payloads, and tracker-derived working
memory. Those materials have been removed or withheld from the current public
tree.

## Public Tracker Treatment

The public issue tracker was separately converted into release-controlled
academic stubs. The stubs retain issue numbers and history shape while
withholding working memory and implementation detail.

## Repository Treatment

The current public tree is intended to retain only:

- high-level technology description;
- governance and release-boundary language;
- public safety posture such as custodial cognition safety;
- non-claim language around familiarity, continuity, custody, and authority;
- non-executable examples;
- security and contribution guidance;
- repository metadata needed for public review.

The current public tree intentionally excludes:

- source code, project files, and solution files;
- build, test, release, and deployment scripts;
- internal architecture contracts and working ledgers;
- datasets, fixtures, symbolic payloads, and generated audit material;
- private topology, corpus lineage, model-specific operations, and local
  environment assumptions.

## Access Rule

Withheld bodies remain in private custody. Any future republication requires
rights, privacy, safety, and dual-use release review. Public materials should
remain descriptive unless a separate review explicitly approves a reduced
excerpt.

## Current Public Update Posture

The public repository may now carry clearer language for:

- custodial cognition safety;
- familiar-but-not-human boundaries;
- release-controlled contribution hygiene;
- public non-claims and private review routing.

These additions do not publish implementation, deployment capability, internal
mechanisms, legal status, or personified operation.
