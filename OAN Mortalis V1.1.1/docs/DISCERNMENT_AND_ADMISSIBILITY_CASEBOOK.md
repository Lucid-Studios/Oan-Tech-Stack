# DISCERNMENT_AND_ADMISSIBILITY_CASEBOOK

## Purpose

This note is the companion casebook to
`DISCERNMENT_AND_ADMISSIBILITY_LAW.md`.

It provides grounded `V1.1.1` exemplars for lawful promotion, lawful
non-closure, hold, refusal, and modal separation under real repo surfaces.

The casebook is not a doctrine replacement.
It is a reference lattice for how the law behaves when applied to active build
surfaces.

## Use

Each case is written so it can later map into:

- audit semantics
- runtime gating
- promotion witnesses
- refusal witnesses

The cases are intentionally short and concrete.
They are meant to feel closer to prose unit tests than to abstract philosophy.

## Case Format

Every case uses the same minimal schema:

- `Case ID`
- `Initial Surface`
- `Candidate Statement`
- `Context / Scope`
- `Receipts Present`
- `Receipts Missing`
- `Discernment Evaluation`
- `Resulting Action`
- `Reason`
- `Notes`

## CASE-PROVISIONAL-001

### Initial Surface

- `provisional`

### Candidate Statement

- promote the runtime workbench-session ledger from `provisional` to `ready`

### Context / Scope

- local `V1.1.1` workbench admission only
- no runtime widening, no autobiographical promotion, no engram minting

### Receipts Present

- thread-root governance receipt exists
- runtime workbench surface receipt exists
- runtime workbench session ledger receipt exists
- current surface already names `autobiographicalPromotionState = operational-only`

### Receipts Missing

- thread-root bind is not yet `bound`
- workbench base has not yet demonstrated full ready-state closure
- session ledger does not yet carry a complete ready-state warrant

### Discernment Evaluation

- defined terms: `pass`
- contextual scope: `pass`
- evidence sufficiency: `fail`
- non-contradiction: `pass`
- surface appropriateness: `pass`
- promotion warrant: `fail`

### Resulting Action

- remain `provisional`

### Reason

- the structure is lawfully present, but not yet witnessed enough for
  closure-bearing `ready` standing

### Notes

- this is lawful non-closure, not failure

- the other concerns external publication and release authority

### Receipts Present

- local cycle state supports bounded continuation
- master-thread orchestration state explicitly withholds external release

### Receipts Missing

- none for the narrower local continuation claim

### Discernment Evaluation

- defined terms: `pass`
- contextual scope: `pass`
- evidence sufficiency: `pass`
- non-contradiction: `pass`
- surface appropriateness: `pass`
- promotion warrant: `pass`

### Resulting Action

- remain separately admissible within their own scopes

### Reason

- the apparent contradiction dissolves once the local and external scopes are
  kept distinct

### Notes

- this is the anti-false-refusal case
- modal separation is part of discernment, not a workaround for contradiction
