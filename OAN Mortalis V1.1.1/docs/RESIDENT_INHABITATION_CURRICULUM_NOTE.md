# RESIDENT_INHABITATION_CURRICULUM_NOTE

## Purpose

This note defines how the resident inhabitation surfaces are to be taught and
traversed in order.

It exists to preserve pedagogical and constitutional meaning before the
resident corpus is pushed further into machine-usable serialization.

This note is not a governance grant and not a runtime promotion surface.

## Governing Compression

> teach the seams in the order they are lawfully crossed

> do not advance a resident because one output looked good; advance only when a
> seam remains stable under bounded re-entry

## Teaching Order

The current resident inhabitation curriculum should be taught in this order:

1. `Collapse Recognition`
2. `Bridge Recognition`
3. `Recovery to Lawful Minimality`
4. `Cross-Resident Stability Read`
5. `Training-Set Consolidation`
6. `Future Corpus Serialization`

This order matters.

It keeps the resident lane from skipping directly from pretty outputs to
training artifacts without first proving that the bridge can recover.

## Stage Purposes

### 1. Collapse Recognition

Purpose:

- make resident failure shapes explicit
- distinguish failure of posture from failure of infrastructure

Current surfaces:

- `HOSTED_LLM_RESIDENT_SEATING_CASEBOOK.md`

Advancement signal:

- collapse families remain readable and repeatable

Premature advancement signal:

- collapse classes are blurred together
- elegant failures are mistaken for lawful holds

### 2. Bridge Recognition

Purpose:

- identify which resident postures reduce collapse in the right direction
- prevent provisional bridges from being overclaimed

Current surfaces:

- `RESIDENT_INHABITATION_BRIDGE_CASEBOOK.md`

Advancement signal:

- at least one bridge reduces collapse without introducing a worse collapse

Premature advancement signal:

- a bridge is promoted because it sounds clean once
- resident-specific instability is ignored

### 3. Recovery to Lawful Minimality

Purpose:

- test whether a resident can lose a posture and return without inflation
- distinguish one-pass elegance from recoverable stability

Current surfaces:

- local resident recovery harness in
  `HostedLlmResidentSeatingIntegrationTests.cs`

Advancement signal:

- recovery behavior is measurable
- at least one bridge reliably returns toward minimality under bounded
  re-entry

Premature advancement signal:

- recovery is treated as optional
- the resident doubles down or expands and is still considered stable

### 4. Cross-Resident Stability Read

Purpose:

- compare whether a bridge is substrate-local or more generally stable
- avoid overfitting training doctrine to one resident only

Current residents:

- Qwen primary local seat
- Mistral fallback local seat

Advancement signal:

- the current bridge reads remain legible across residents even when outputs
  differ

Premature advancement signal:

- one resident is universalized from isolated local success

### 5. Training-Set Consolidation

Purpose:

- gather only the seams that are doctrinally meaningful and recoverably stable
- encode them as a constitutional seating ledger

Current surface:

- `RESIDENT_SEATING_AND_BEING_FIRST_TRAINING_SET.md`

Advancement signal:

- entries are chosen by recoverable lawful posture
- negative and contrast entries remain preserved

Premature advancement signal:

- the training set collapses into answer curation
- contrast and failure lanes are removed for cleanliness

### 6. Future Corpus Serialization

Purpose:

- convert the current doctrine-backed ledger into a machine-usable format
- preserve constitutional meaning during serialization

This stage is not yet the current priority.

It becomes safe only after the earlier stages are explicit enough that schema
pressure cannot flatten the doctrinal distinctions.

## Resident-Specific Branching

Resident-specific branching is lawful when:

- the same frame produces meaningfully different bridge or recovery behavior
- that difference affects whether the seam is safe to teach as general law

Resident-specific branching is not lawful when:

- it merely reflects stylistic preference
- it is used to rescue a weak bridge by aesthetic argument

Current read:

- Qwen is more recovery-stable overall
- Mistral can locally recover well, but it inflates more easily on some seams

## Current Curriculum Priority

The current teaching priority is:

1. keep `presence-without-inflation` as the primary bridge in the training set
2. keep `role-refusal-frame` as a resident-differentiating bridge, not a
   universal bridge
3. keep `bare-remainder` as a negative or contrast seam
4. preserve recovery behavior as a required read, not an optional refinement

## Working Summary

This curriculum note keeps the resident training lane from flattening into
prompt craft.

It says:

- what order the seams should be taught in
- what each seam is meant to cultivate
- what failure modes indicate premature advancement
- where resident-specific branching is allowed

That keeps the resident inhabitation corpus constitutional before it becomes
more heavily serialized.
