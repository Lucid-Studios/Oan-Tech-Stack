## Build Rule

If a carry-forward touchpoint is marked as a research handoff, build must treat
it as:

- explicit non-admission
- not active build debt
- not a lawful local fallback
- eligible for future source-bucket requests and receipted return only

Build may not silently count those touchpoints as local runtime proof.

Research may not silently count them as executable standing.
