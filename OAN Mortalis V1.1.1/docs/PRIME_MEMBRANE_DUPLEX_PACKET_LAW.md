# PRIME MEMBRANE DUPLEX PACKET LAW

## Purpose

This note defines the first bounded packet family for the `Prime Membrane`
read now active in `V1.1.1`.

It exists to keep four things distinct:

- membrane source
- cryptic projection posture
- bounded `CME` standing
- Prime-side closure eligibility

This note does not grant Prime closure.
It only seats the first lawful membrane-side handoff law that can assess
whether a duplexed field packet may return for explicit Prime-side receipt.

## Core Read

The active stack should now read the Prime/Cryptic relation this way:

- `Prime Membrane` is the lawful source surface of readable presence
- `Cryptic` is the duplex-projected shadow field where cognition may hover,
  rehearse, braid, and ripen before commitment
- bounded `CME` state is a local standing formed from that duplex, not the
  whole field itself
- closure eligibility is not closure and must remain separate from it

In compact form:

> the membrane issues presence, the projection carries participation, the
> bounded state stands locally, and Prime closure remains explicitly receipted.

## Packet Family

The first bounded packet family for this seam is:

- `PrimeMembraneSourcePacket`
- `CrypticProjectionPacket`
- `CmeBoundedStatePacket`
- `DuplexFieldPacket`
- `PrimeClosureEligibilityReceipt`

Together they must answer four questions plainly:

1. where did this arise
2. in what projected posture does it currently stand
3. what bounded state, if any, has formed
4. is it eligible to return for Prime-side closure review

## Projection Postures

The first projected posture family is:

- `Hovering`
- `Rehearsing`
- `Braided`
- `Latent`
- `Ripening`
- `Unresolved`

These are not decorative states.
They preserve that the projected field may remain non-binding while cognition
is still gathering shape.

The stack must not collapse them into a flat active/inactive distinction.

## Bounded Standing

The first bounded `CME` standing family is:

- `None`
- `Candidate`
- `Bounded`
- `Anchored`

This family answers whether a local standing has formed at all, and whether it
has formed strongly enough to be reviewable at the membrane.

It does not itself issue Prime closure.

## Handoff Law

The first membrane-side handoff law is:

- membrane origin must be explicitly attested
- unresolved projection may not return for Prime-side closure review
- non-binding projected postures may remain candidate-bearing without being
  treated as closure-ready
- braided and ripening projected states may become reviewable when bounded
  standing exists
- only ripening projection with anchored bounded standing may become eligible
  for explicit membrane-side receipt

Even there, eligibility is not closure.

The governing brake is:

> no projected or bounded state shall be treated as Prime closure without an
> explicit membrane-side receipt.

## Non-Collapse Rules

The packet family must preserve these boundaries:

- membrane source is not the same thing as projection posture
- projection posture is not the same thing as bounded standing
- bounded standing is not the same thing as Prime closure eligibility
- Prime closure eligibility is not the same thing as Prime closure itself

The packet family therefore exists to prevent two errors:

- projected participation being mistaken for canonical closure
- local bounded standing being mistaken for the whole duplexed field

## Immediate Build Consequence

Any future `RTME`, `SLI`, or `EC` seam in this area should say plainly:

- which membrane issued the packet
- what projected posture the packet currently carries
- whether a bounded `CME` state has formed
- whether the packet is candidate-only, review-required, or eligible for
  explicit membrane-side receipt
- whether later membrane-side interpretation is reading only active projection,
  stable braid, return-candidate history, or defer-only evidence

If it cannot say that plainly, it should remain held in the projected field.
