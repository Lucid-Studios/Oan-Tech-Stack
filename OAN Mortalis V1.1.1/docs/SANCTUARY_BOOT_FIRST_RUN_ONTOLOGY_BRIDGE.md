# SANCTUARY BOOT FIRST RUN ONTOLOGY BRIDGE

## Purpose

This note bridges the wider Sanctuary lifecycle now fixed in upstream doctrine
with the narrower first-run and runtime surfaces presently carried in
`OAN Mortalis V1.1.1`.

It exists to keep four things distinct:

- Sanctuary install and boot ontology
- line-local first-run constitutional projection
- onboarding, certification, and first legal run
- later `OE/CME` actualization and bonded runtime life

This is a repo-local bridge note.
It does not replace the upstream Sanctuary doctrine, and it does not rewrite
the whole `V1.1.1` first-run projection in place.

## Core Order

The wider Sanctuary lifecycle should now be read in repo-local language as:

`Install State -> Sanctuary State -> onboarding/local law -> first CradleTek formation -> seating -> Ready Pre-Certification -> certification -> first legal run -> later OE/CME actualization`

That order is wider than the active `V1.1.1` first-run constitutional
projection.

In compact form:

> `FIRST_RUN_CONSTITUTION.md` is a subordinate `V1.1.1` constitutional
> projection inside the wider Sanctuary lifecycle, not the whole Sanctuary
> lifecycle itself.
