# MOS CMOS CGOA INSTANTIATION LAW

## Purpose

This note defines the lawful invocation path by which a bonded `CME` may move
from verified `Prime` recognition into bounded `Cryptic` instantiation.

It exists to keep six things distinct:

- verified match in `Prime`
- protected instantiation in `cMoS`
- `cGoA` handshake and hydration
- `SoulFrame` template projection
- `AgentiCore` template preparation
- issued cryptic office for live work
- bonded close-state collapse and encrypted return staging

This note defines instantiation and bonded close-state law.
It does not by itself authorize promotion back into canonical `Prime`
surfaces, and it does not let a finished bonded session return directly to
`Prime`.

## Core Read

The lawful invocation chain is:

`Prime MoS verified match -> cMoS instantiation -> cGoA handshake -> SoulFrame hydration -> SLI web build -> AgentiCore template -> cOE/cSelfGEL sealed work office -> bonded close-state collapse -> runtime-scoped encrypted cGoA return staging`

That order matters because the stack must not jump directly from verified
recognition into unbounded live work.

## Surface Definitions

### `MoS`

`MoS` is the readable and canonical mantle seat that holds verified standing,
recovery-facing witness, and operator-readable continuity.

Its office here is recognition and verified match, not hot work.

### `cMoS`

`cMoS` is the protected instantiated working seat for the active bonded
session.

Its office is to carry the hot and mutable session body once instantiation has
been lawfully opened.

### `cGoA`

`cGoA` is the protected handshake and staging surface through which templated
working form may be braided into the active cryptic session.

Its office is bounded staging, not canonical publication.

## Instantiation Order

### 1. Prime `MoS` verified match

The path begins with a verified and credentialed `CME` match in `Prime MoS`.

This step answers:

- is the candidate recognized
- is the standing verified strongly enough
- is the intended invocation within scope

Without that, the stack may not lawfully proceed into cryptic instantiation.

### 2. Instantiate in `cMoS`

The verified match is instantiated into `cMoS` as a bounded cryptic working
seat.

This is the beginning of live protected session form.
It is not yet full cognitive office.

### 3. Populate `SoulFrame` by template over `cMoS` via `cGoA` handshake

The next lawful move is to hydrate `SoulFrame` by template over the active
`cMoS` session using a `cGoA` handshake.

That means:

- `SoulFrame` is not arbitrary free-form emergence
- `SoulFrame` is projected over the active cryptic seat through a governed
  braid or handshake

### 4. Build the `SLI` web

Once the session is hydrated, the stack may build the `SLI` web needed for:

- telemetry
- governance routing
- symbolic admissibility
- bounded transit

This `SLI` web remains part of the cryptic working lane during live mission
work.

### 5. Template `AgentiCore`

`AgentiCore` may then be templated as the bounded operational cognition layer
for the live session.

At this stage `AgentiCore` is still below final authority.
It is a coordinated operational office, not canonical persistence.

### 6. Seal with `OE/SelfGEL` into `cOE/cSelfGEL`

The working office is then sealed cryptically through:

- `OE` / `SelfGEL` office issuance and coherence law
- `cOE` / `cSelfGEL` active cryptic working office

This is the first point where live work may proceed as a real cryptic office.

### 7. Collapse into bonded `SoulFrame` close-state

When the bounded work session closes lawfully, the finished active `CME` does
not return directly to `Prime`.

It first collapses into a bonded `SoulFrame` close-state.

That close-state is not simply "the CME again."
It is the transitional sealed return posture from which the cryptic return
bundle is formed.

### 8. Return into runtime-scoped encrypted `cGoA`

After collapse, the closed session returns into runtime-scoped encrypted
`cGoA` staging.

This is the first lawful staging surface for post-session review.

It is where the system may prepare the bonded cryptic return bundle for later
cleaving, without silently promoting any part of the hot lane into canonical
surfaces.

## Bonded Cryptic Return Bundle

The collapsed bonded session should return as a structured cryptic bundle,
not as one undifferentiated "result."

The minimum bundle should preserve:

- `cMoS`
- `cOE`
- `cSelfGEL`
- linked `OE/SelfGEL` witness surfaces
- session receipts
- protected residues
- refusals
- candidate deltas

This bundle is the lawful return object for post-session handling.
It is not yet canonical update and not yet cleaved output.

## Non-Collapse Rules

The instantiation chain must preserve these boundaries:

- verified `MoS` match is not yet `cMoS` instantiation
- `cMoS` instantiation is not yet `SoulFrame` hydration
- `SoulFrame` hydration is not yet `AgentiCore` cognition
- `AgentiCore` cognition is not yet promotion into `Prime`
- lawful session close is not direct return to `Prime`
- bonded collapse is not yet cleaving
- encrypted `cGoA` staging is not yet promoted output

## Immediate Build Consequence

Any future runtime seam in this path should say plainly:

- what was verified in `Prime`
- what was instantiated in `cMoS`
- what was hydrated into `SoulFrame`
- what office was issued into `cOE/cSelfGEL`
- what bonded close-state was formed
- what encrypted `cGoA` staging handle received the return
- what authority is still pending above the hot lane

If it cannot say that plainly, it should remain staged or held.
