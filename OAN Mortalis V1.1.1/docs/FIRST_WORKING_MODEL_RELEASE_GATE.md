# FIRST_WORKING_MODEL_RELEASE_GATE

## Purpose

This note defines the first lawful admissibility gate for calling the current
system a first working model.

It is not a feature list.
It is not a momentum marker.
It is the place where the stack must justify its existence as a working model
under its own law.

## Governing Compression

The governing rule is:

> the first working model is not declared by momentum. It is admitted only
> when active runtime truth, release-form sibling body, and read-only witness
> surfaces justify the claim together.

That means:

- `V1.1.1` supplies the active executable truth and runtime-bearing proof
- `V1.2.1` supplies the release-form sibling body and production-shaping truth
- witness surfaces may observe and summarize, but they do not self-authorize
  readiness
- the gate must be able to `pass`, `hold`, or `fail`

No "mostly ready" reading is lawful here.

Public release, readiness, tag, milestone, and status wording is governed by
`PUBLIC_RELEASE_READINESS_WORDING_LAW.md`.

Readiness may be witnessed only to the level currently supported by repo-local
evidence.

## Gate Status Grammar

### `pass`

Use `pass` only when:

- all required evidence exists
- no blocking contradiction remains
- the line-role split is still truthful

### `hold`

Use `hold` when:

- the direction is correct
- the existing body is lawful
- but one or more required seams are still intentionally incomplete

### `fail`

Use `fail` when:

- evidence contradicts the claim
- required runtime truth breaks
- required release-form truth breaks
- witness surfaces expose active dishonesty or invalid standing claims

## Non-Blocking Withheld Seams

The following may remain withheld without invalidating first-working-model
admissibility by themselves:

- later operator realization
- thicker `V1.2.1` emitted telemetry inventory
- the future `V1.2.1` test graph
- final single-line promotion or archive of `V1.1.1`

These are later gates.
They are not this gate.

## Blocking Conditions

The first working model must remain at least `hold` and may not `pass` if any
of the following remain true:

- `V1.1.1` build, test, or hygiene is not green
- `V1.2.1` is not buildable as a sibling release-form body
- the role split between active runtime truth and release-form sibling body is
  blurred or contradicted
- witness tooling cannot report both lines truthfully
- the working-model claim requires a seam that is still explicitly withheld
