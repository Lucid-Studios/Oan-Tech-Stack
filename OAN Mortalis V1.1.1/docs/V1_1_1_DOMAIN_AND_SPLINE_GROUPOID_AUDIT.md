# V1_1_1_DOMAIN_AND_SPLINE_GROUPOID_AUDIT

## Purpose

This note audits the active `V1.1.1` documentation body and condenses it into
one categorical read.

It exists to stop the current doc set from drifting as:

- free-floating doctrine with no primary owner
- lower-runtime notes claiming parent-chamber office
- lifecycle notes smearing install, build, run, rest, and exit together

This is an audit and condensation note.
It does not replace the underlying law notes.

## Governing Compression

The current `V1.1.1` doc body should be read through two orthogonal groupoids:

- domain:
  `Sanctuary -> CradleTek -> SoulFrame -> AgentiCore`
- spline:
  `Install -> Build -> Run -> Rest -> Exit`

The domain answers:

- who owns the office named by the note

The spline answers:

- when in the line-lifecycle that note is primarily acting

Every active build-facing note should now be readable as:

- one primary domain
- one primary spline

`SLI` and line-governance notes currently condense upward into `Sanctuary` for
document ownership in `V1.1.1`, even when they later project into lower
runtime families.

The emitted telemetry body now also has a dedicated end-to-end classification
surface in `TELEMETRY_BUNDLE_AND_GROUPOID_TAXONOMY.md`.

That telemetry note does not replace this domain/spline placement audit.
It extends the reading of emitted telemetry surfaces with additional
semantic/authority/continuity/retention/package groupoids.

## Audit Result

The active doc body now condenses cleanly enough to use this matrix as the
default reading order.

The main findings are:

- most crosscutting build, automation, admission, topology, and translational
  notes belong to `Sanctuary`, not to an ambient stack layer
- `CradleTek` notes currently cluster at the live invocation, workbench, and
  bounded runtime-control seam
- `SoulFrame` notes currently cluster around steward witness, binding, seal,
  collapse, and return preparation
- `AgentiCore` remains the late-path cognition domain and should not absorb
  higher governance, issuance, or promotion authority
- `Exit` remains intentionally thin in `V1.1.1`; the line has one true exit
  note and should not multiply them casually

Audited active docs: `49`

Condensed primary-domain counts:

- `Sanctuary`: `35`
- `CradleTek`: `2`
- `SoulFrame`: `8`
- `AgentiCore`: `4`

- install and deployment boundary

### `SoulFrame`

`SoulFrame` presently owns the bridge, binding, seal, and collapse seam.

It presently owns:

- Steward-witnessed office approach
- elemental binding
- actualization-seal uptake
- bonded close-state and return collapse preparation

### `AgentiCore`

`AgentiCore` presently owns the late-path living cognition seam.

It presently owns:

- living `AgentiCore` packet and hold law
- `Listening Frame`
- `Compass`
- bounded cognition coordination

## Spline Read

### `Install`

Use `Install` when a note governs:

- migration into the line
- boot, deployment, or topology footing
- first-run constitutional projection
- initial packet formation before wider runtime life

### `Build`

Use `Build` when a note governs:

- readiness
- carry-forward
- packet hardening
- orchestration and admission planning
- active build automation and work publication

### `Run`

Use `Run` when a note governs:

- active runtime law
- bounded invocation
- membrane crossing
- duplex live work
- live cognition coordination

### `Rest`

Use `Rest` when a note governs:

- end-state transition
- hold
- human review
- report consumption
- bonded collapse and return audit

### `Exit`

Use `Exit` when a note governs:

- retirement
- line close
- release-ending judgment

## Condensation Consequences

Use this audit as the current default when later notes are added or revised.

The practical rules are:

- if a note is crosscutting and no lower family owns it clearly, place it in
  `Sanctuary`
- if a note fixes end-to-end telemetry classification across lanes, place it in
  `Sanctuary / Build` even when its emitted artifacts later span `Run` and
  `Rest`
- if a note governs first-working-model admissibility, line-role split, and
  evidence binding across sibling lines, place it in `Sanctuary / Build`
- if a note governs runtime invocation, workbench control, or bounded cryptic
  instantiation, place it in `CradleTek`
- if a note governs bridge, binding, seal, close-state, or return collapse,
  place it in `SoulFrame`
- if a note governs living cognition posture, `Listening Frame`, `Compass`, or
  bounded cognition coordination, place it in `AgentiCore`
- do not let `AgentiCore` absorb admission, issuance, duplex, or canonical
  promotion law
- do not let `Install` or `Exit` notes proliferate without explicit line-law
  need

For emitted telemetry surfaces, use this audit for document placement and use
`TELEMETRY_BUNDLE_AND_GROUPOID_TAXONOMY.md` for semantic carrier,
authority-class, continuity-class, retention-class, and package-class reading.

This note is the current categorical condensate for the active `V1.1.1`
documentation body.
