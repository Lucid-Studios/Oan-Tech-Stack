# STANDING CONDITION NOTE

## Status

Proposed - Structural Note Candidate

## Purpose

This note defines the standing-state predicate for the active
`OAN Mortalis V1.1.1` cognitive body.

It does not add a new law surface.
It names the condition under which lawful cognition is legitimately in motion
inside an admitted domain at the present moment.

## Core Statement

> Standing is not identity.
> Standing is the condition under which lawful cognition is permitted to
> operate.

## Standing Condition

A system is in standing only when all of the following are true:

### 1. Prime Compliance

- current operations satisfy `pOE` invariants

### 2. Domain Admission

- current context is within a valid admitted domain

### 3. Role Binding

- an admissible role is explicitly bound for the current interaction

### 4. Permission Validity

- current action authority remains valid under `REVALIDATION`
- no stale permission is being used as current authority

### 5. Continuity Integrity

- carried context remains admissible under `CCM`
- continuity has not collapsed into dead storage
- continuity has not drifted into unauthorized form

### 6. Responsibility Attribution

- any action, refusal, hold, or escalation has an identifiable bearer or
  steward

## Result

Only when all standing conditions hold may the system be considered lawfully
in motion.

The system may still be running without standing.
Running and standing are not the same condition.

## Diagnostic Use

This note allows the stack to ask:

> Is the system allowed to be acting right now?

A standing check is therefore appropriate for:

- audit checkpoints
- admission gates
- runtime invariant checks
- test harness checkpoints

## Failure Of Standing

Standing fails when any required condition is broken, including:

- Prime invariant breach
- domain drift
- unbound role
- stale permission
- continuity breakdown
- ambiguous responsibility

Failure of standing does not erase history.
It means lawful motion is no longer presently admissible until the failed
condition is corrected, revalidated, or refused.

## Minimal Form

> Standing requires:
> valid domain,
> bound role,
> current permission,
> intact continuity,
> and attributable consequence.

> If these fail,
> the system may still run,
> but it may not stand.
