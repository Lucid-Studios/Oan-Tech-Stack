# ASSIMILATION_RECEIPT_AND_DELTA_BRIDGE_NOTE

## Purpose

This note defines the smallest lawful bridge from admission into later
receipt-governed change.

It exists so later `GEL` and `SelfGEL` growth can remain traceable, auditable,
and non-ambiguous without pretending that the full `Delta` engine already
exists in runtime.

This slice is doctrine-only for the bridge.

## Governing Compression

> assimilation receipts govern later `Delta`

Admission is not enough on its own.

The system must later know not only whether something was admitted, but how it
entered and how conflict was resolved.

## Assimilation Set

The current minimal assimilation set is:

- `new`
- `instance`
- `refine`
- `extend`
- `relate`
- `reject`

Current reading:

- `new`
  - introduces previously unseen lawful structure
- `instance`
  - strengthens an existing structure without redefining it
- `refine`
  - sharpens an existing boundary
- `extend`
  - adds new lawful structure without mutation
- `relate`
  - connects existing structures without identity collapse
- `reject`
  - is not admitted

## Conflict Outcome Set

The current minimal conflict outcome set is:

- `coexist`
- `strengthen`
- `compose`
- `replace`
- `scope_split`
- `decompose`
- `refuse`

These outcomes are enough to keep later growth from flattening into silent
overwrite.

## Receipt Fields As Delta Predicates

The current receipt fields that predicate later `Delta` are:

- assimilation type
- anchor lineage
- layer
- boundary/context witness
- integrity seal
- conflict-resolution outcome

Compressed:

> receipt fields govern how later change is allowed to occur

## GEL And SelfGEL Coupling

The same receipt later drives different but coupled change in the two
substrates:

- `GEL`
  - structural change
- `SelfGEL`
  - continuity and identity-trace change

Compressed:

> `GEL` and `SelfGEL` later receive coupled but non-identical effects from the
> same receipt

That distinction matters.

It prevents shared structure from being collapsed into personal continuity, and
it prevents personal continuity from silently rewriting shared structure.

## Deferred Boundary

This slice does not define:

- executable `Delta` accumulation
- executable condensation
- executable stability-metric thresholds
- skill condensation

Accumulation, condensation, and stability metrics remain deferred.

## Minting Boundary

Bonded minting remains deferred until after this substrate is seated.

This bridge is part of the lawful substrate that later minting certification
must stand on.

It is not the minting act itself.

## Working Summary

This note keeps later change honest:

- admission remains distinct from transformation
- assimilation type remains distinct from conflict outcome
- receipt remains the governing bridge into later `Delta`
- coupled change remains distinct across `GEL` and `SelfGEL`

without pretending that the runtime accumulation or condensation engines
already exist.
