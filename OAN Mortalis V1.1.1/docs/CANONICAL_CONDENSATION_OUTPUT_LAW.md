# Canonical Condensation Output Law

## Purpose

This note defines the first canonical condensation output for later lawful
reuse.

The canonical condensation output is doctrinally read as:

- `Pi_c` Prime canonical face
- `Lambda_c` Cryptic canonical face
- `R_c` condensation receipt
- `U_c` usage contract

## Cryptic Canonical Face

- `event-id`

- `result = condensed`

## Usage Contract

The usage contract carries:

- `allowed-ops`
- `layer-constraints`
- `posture-req`
- `scope`
