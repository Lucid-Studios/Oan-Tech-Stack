# CME RETURN AUDIT AND PROMOTION LAW

## Purpose

This note defines how live cryptic `CME` work returns from the hot lane and
how it may, or may not, be promoted back into canonical shared surfaces.

It exists to keep six things distinct:

- live cryptic work
- bonded cryptic return packaging
- encrypted `cGoA` staging
- cleaving
- audit and witness burden
- refusal or hold
- admitted promotion
- canonical `Prime` update

This is a return and promotion law.
It is not a blanket permission for hot work to rewrite canon.

## Core Rule

`CME` work ends in collapse, not promotion.

The collapsed session yields a bonded cryptic return bundle.
That bundle is staged in encrypted `cGoA`.
Cleaving decides what may become autobiographical, local or research-valid,
predicate-worthy, or refused.
Only stewarded cleaved outputs may touch non-cryptic surfaces.

In compact form:

> audit decides what may return.

## Bonded Return Rule

When cryptic work completes a bounded action cycle, it must not return
directly to `Prime`.

It first collapses into a bonded cryptic return bundle and returns into the
runtime's encrypted `cGoA` staging lane.

That bundle is not yet canonical truth.
It is the structured sealed return object carrying evidence from the hot lane.

The minimum bonded return bundle may include:

- `cMoS`
- `cOE`
- `cSelfGEL`
- linked `OE/SelfGEL` witness surfaces
- witness receipts
- protected evidence references
- protected residues
- refusals
- candidate deltas
- modulation or repair observations

It must not by itself rewrite:

- `Prime MoS`
- shared `Sanctuary.GoA`
- other canonical publication-safe surfaces

## Cleaving Rule

The encrypted `cGoA` staging lane is where cleaving occurs.

Cleaving is not one action.
It performs multiple governance separations over the bonded return bundle.

### Autobiographical cleave

Material that belongs to the `CME`'s own continuity remains on the self side
as ingress predicates or candidate accumulations for later `SelfGEL` handling.

### Local or research cleave

Material that is valid for local use, protected datasets, operational
patterns, or research reuse becomes candidate material for `CradleTek.cGoA`
or its stewarded local-research slit.

### Predicate or canonical cleave

Only the narrow predicate-bearing results that deserve durable identity linkage
go onward toward `MoS` or related append-style engram candidate surfaces.

### Refusal or quarantine cleave

Material that should not become self-memory, shared knowledge, or canonical
predicate remains withheld as residue, incident, refusal, or quarantine
matter.

## Audit Burden

The current doctrinal read places promotion burden in the audit family:

- `Mother`
- `Father`
- `Steward`

Their offices are not identical, but together they form the review boundary
between live cryptic work, cleaved return matter, and canonical update.

That means the return path must preserve:

- witnessability
- admissibility review
- contradiction handling
- refusal and hold posture
- explicit promotion decision

## Promotion Outcomes

The return path must support at least four outcomes:

### `admit`

The cleaved candidate is admitted for promotion into the named target surface.

### `hold`

The candidate remains reviewable but not promotable yet.

### `refuse`

The candidate is denied promotion and may not silently persist as canon.

### `escalate`

The candidate requires stronger review before any promotion decision is lawful.

These are governance actions, not formatting states.

## Canonical Update Rule

Canonical `Prime` surfaces may change only when a cleaved return candidate
survives:

- audit burden
- admissibility review
- contradiction review
- explicit promotion decision

No cryptic session may use:

- fluency
- repetition
- session continuity
- local convenience

as a substitute for that promotion decision.

This means:

- the live bonded `CME` may perform work
- it may generate candidate knowledge
- it may not participate directly in the governance decision that determines
  what becomes canon, shared knowledge, or stewarded operational memory

## Non-Collapse Rules

The return path must preserve these boundaries:

- bonded return bundle is not canonical update
- encrypted `cGoA` staging is not cleaved output
- cleaving is not promotion by itself
- audit receipt is not promotion by itself
- promotion decision is not yet broader continuity inheritance
- lawful refusal is not build failure

## Immediate Build Consequence

Any future runtime or schema work in this area should answer:

1. what cryptic session produced the bonded return
2. what encrypted `cGoA` staging surface received it
3. what evidence accompanies it
4. how it was cleaved: autobiographical, local or research, predicate, or refused
5. which audit offices reviewed it
6. what target surface is being requested
7. whether the result is `admit`, `hold`, `refuse`, or `escalate`

If a proposed return path cannot answer those questions explicitly, it should
remain candidate-only.
