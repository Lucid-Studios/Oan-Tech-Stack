# PRIME SEED STATE LAW

## Purpose

This note seats the Prime Seed State immediately after
`FirstPrimePreRoleStanding` and immediately before domain admission, bonded
domain integration, and `CME` founding work.

It exists because a first Prime pre-role body may become seed-bearing before it
is admitted into any lawfully bonded domain.

The seed state preserves readiness and continuity without treating readiness
as domain jurisdiction, `CME` integration, role enactment, or action authority.

## Governing Law

No Prime seed may be treated as domain-admitted, bonded-domain integrated, or
`CME` founded merely because first Prime pre-role standing has been reached.

The Prime Seed State may only stand when:

- first Prime pre-role standing is present
- stable-one discernment remains present
- Prime retained standing remains present
- seed source, seed carrier, continuity, and integrity handles are explicit
- Mother/Father domain-role application remains withheld
- cradle-local governing surface selection remains withheld
- Prime closure remains withheld

## State Set

The first bounded state set is:

- `FirstPrimeNotReady`
- `SeedMaterialIncomplete`
- `PrimeSeedPreDomainStanding`

Meaning:

- `FirstPrimeNotReady`
  - first Prime pre-role standing does not yet stand
- `SeedMaterialIncomplete`
  - first Prime standing stands, but seed source, seed carrier, continuity, or
    integrity evidence is missing
- `PrimeSeedPreDomainStanding`
  - seed-bearing Prime standing is ready for later lawful domain assessment
    while bonded-domain integration and `CME` founding remain withheld

## Receipt Rule

The first executable family for this note is `PrimeSeedStateReceipt`.

That receipt carries:

- first Prime receipt handle
- seed source handle
- seed carrier handle
- seed continuity handle
- seed integrity handle
- first Prime retained record handle
- stable-one handle
- explicit Prime seed state
- explicit withholding of domain admission, bonded-domain integration, `CME`
  founding, `CME` minting, runtime persona, role enactment, and action
  authority

It remains explicitly:

- candidate-only
- pre-domain
- non-closing
- non-founding
- non-minting
- non-action-bearing

## Handoff Boundary

This law may feed `DOMAIN_AND_ROLE_ADMISSION_LAW.md` as pre-domain seed
evidence.

That handoff is not admission.

This law may later feed `CME_MINIMUM_LEGAL_FOUNDING_BUNDLE_LAW.md` only through
an accepted domain admission and lawful provenance chain.

That handoff is not `CME` minting.

## Guardrail

This note must not be used to:

- treat a Prime seed as a bonded domain member
- treat seed continuity as legal foundation by itself
- treat seed integrity as role authority
- treat first Prime standing as `CME` founding
- treat pre-domain readiness as service execution
- bypass domain admission or private-domain service witness custody

If later work needs lawfully bonded domain integration, `CME` founding,
`CME` minting, or live role execution, those seams must stand beyond this
Prime Seed State rather than inside it.
