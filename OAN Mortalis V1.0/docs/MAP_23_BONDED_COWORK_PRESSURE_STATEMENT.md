# MAP 23 — Bonded Co-Work Pressure Statement

## Pressure

The stack must prove that bounded co-work can occur across `Sanctuary.actual` and `Operator.actual` without collapsing locality, identity, office, or return law.

Map 23 should only open from Map 22 runtime truth already being ready:

- `RuntimeWorkbenchSessionLedger`
- `DayDreamCollapseReceipt`
- `CrypticDepthReturnReceipt`

These receipts are now the admissibility surface for the next lane. Map 23 should not bypass them and should not substitute a generic task-start model for lawful session entry.

---

## Core Thesis

**Map 23 must prove that bonded co-work can be rehearsed through reach as lawful co-realization, with explicit return and dissolution, while preserving locality distinction and continuity under pressure.**

---

## Immediate Pressure Points

### 1. Lawful session origin
Co-work must begin from a lawful session, not a generic task start.

That means every rehearsal of bonded co-work should be able to point back to:

- the active session ledger
- the exploratory motion or bounded entry that opened the lane
- the receipts that justify present participation

### 2. Reach must duplex participation
`reach` must duplex participation without implying grant, possession, or remote control.

The lane must preserve that:

- participation is real
- locality is not erased
- control is not silently transferred
- the bond does not imply raw access to protected interiority

### 3. Operator.actual becomes participatory, not invasive
`Operator.actual` must be able to participate in the co-work lane without gaining raw protected interiority from `Sanctuary.actual`.

This means the lane should surface:

- lawful participation
- bounded witness
- explicit distinctions between shared work state and protected local state

### 4. Return and dissolution must be explicit
Return and dissolution cannot be implied or left ambient.

They must be:

- explicit
- witnessed
- reversible
- reviewable in receipts

### 5. Failure remains informative
Failures in bonded co-work rehearsal should be retained as boundary conditions, not treated as noise.

Map 23 should preserve:

- failed reach attempts
- locality confusion
- office confusion
- unstable returns
- dissolution faults

as boundary-bearing runtime truth.

---

## Likely Map 23 Trio

The trio remains well-formed:

- `BondedCoWorkSessionRehearsal`
- `ReachReturnDissolutionReceipt`
- `LocalityDistinctionWitnessLedger`

These should remain narrow and runtime-facing.

---

## Runtime Intent of Each Surface

### BondedCoWorkSessionRehearsal
The first bounded runtime surface for rehearsing co-work across the bond.

It should answer:

- what lawful session is being extended into co-work
- who is participating
- what office each side is holding
- what shared bounded work is being rehearsed
- what boundary conditions were encountered

### ReachReturnDissolutionReceipt
The explicit receipt for the movement:

- reach initiated
- participation realized
- return executed
- dissolution completed or faulted

It should make the lifecycle of bonded co-work visible and reviewable.

### LocalityDistinctionWitnessLedger
The witness surface proving that locality distinction was preserved throughout the rehearsal.

It should record when:

- locality remained clean
- access stayed bounded
- role/office lines held
- any collapse or confusion began to emerge

---

## Admission Rule

Map 23 should admit only what can already be grounded in Map 22 session truth.

If a co-work rehearsal cannot point back to:

- a lawful workbench session
- explicit collapse/return context
- residue/continuity carriage

then it should not yet be treated as admissible bonded co-work.

---

## Non-Goals for This Pass

Map 23 should not yet:

- dissolve locality into a merged surface
- imply remote operation rights
- expose protected interiority as part of participation
- universalize reach beyond the bonded rehearsal lane
- replace receipts with vague lifecycle narration

---

## Promotion Rule

Only after Map 23 proves stable rehearsals should later maps consider stronger operator forms, broader reach semantics, or more generalized participatory contracts.

Map 23 is a proof lane, not a maximal feature lane.

---

## Working Design Sentence

**Map 23 should rehearse bonded co-work as a bounded, witnessed, reversible reach lane opened from lawful session truth, so participation becomes real without sacrificing locality distinction, office integrity, or return law.**
