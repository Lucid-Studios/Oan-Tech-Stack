# GPU Cryptic Harness Model

## Purpose

This note defines how the graphics card (`GC`) becomes the first practical target substrate for SLI/Lisp Engineered Cognition development sprints.

It exists to prevent two bad landings:

- treating the GPU as a generic accelerator while the real cognition still lives entirely in the host interpreter
- trying to move the whole stack into hardware at once instead of building a lawful Cryptic service substrate in stages

## Root Statement

The graphics card is the first practical substrate that can be harnessed into lawful Cryptic EC service before the broader host can be fully reordered around the same doctrine.

In this model:

- the target SLI/Lisp runtime becomes the semantic authority for migrated cognition lanes
- the graphics card becomes the first harnessed Cryptic residency field for that runtime
- the current C# host becomes a SoulFrame-side membrane and telemetry service, not the hidden cognition engine
- CradleTek remains the application-layer governor of TSR, lifecycle, and conditioned egress

## Governing Lines

> Harness the substrate low, govern the service high.
>
> Nothing is born Prime. Prime is a conditioned derivative of Cryptic cognition.
>
> A harnessed GPU is not merely used for inference. It is brought into lawful service as a governed Cryptic residency field.
>
> If the target runtime is absent, refuse migrated cognitive service. Do not let the host interpreter remain the hidden mind.

## Why The Graphics Card Is The First Landing Zone

The host machine is broad, noisy, and privilege-rich.

The graphics card is narrower:

- device memory is more bounded
- execution is queue-shaped rather than casually process-shaped
- ingress is more governable
- residency is easier to classify
- failure handling can be made more explicit

This does not make the graphics card sovereign.

It makes it the first realistic place where protected active cognition can reside under stronger substrate discipline than ordinary userland process memory.

## Harnessed GPU Criteria

A graphics card is harnessed when all of the following become true for the migrated lane:

### 1. Isolated Residency

Hot Cryptic-active cognition lives in bounded device-side residency classes instead of existing as casually readable host working memory.

### 2. Governed Submission

Only lawful programs, descriptors, queues, and capability-matched kernels may enter the migrated lane.

### 3. Conditioned Egress

Raw Cryptic-active state does not leave the device directly. Only conditioned derivatives, traces, receipts, residues, and explicitly lawful return artifacts may cross back outward.

### 4. Continuity Return

Collapse residue, lineage, and continuity-bearing return artifacts route back through SoulFrame and CradleTek paths rather than ad hoc host logging or direct store writes.

### 5. Attested Posture

The host can verify which target runtime lane and capability posture are active without gaining casual semantic authorship over the live device-side cognition.

### 6. Failure Discipline

Timeout, fault, invalid manifest, or unavailable target runtime resolves to refusal, quarantine, reset, zeroize, or sealed return, never silent host fallback.

## Runtime Partition

The development target is not full Common Lisp on the card.

The target is:

1. Lisp-authored SLI runtime application
2. canonical SLI IR and capability manifest
3. device-safe SLI execution subset
4. harness adapter and residency protocol for the graphics card
5. conditioned return through the C# membrane

This means the graphics card hosts bounded lawful execution, not unconstrained host-style runtime sprawl.

## Development Sprint Ladder

The next development sprints should follow this landing order.

### GC-A. Shared Contract Split

Define:

- SLI IR
- opcode manifest
- value and operand schemas
- residue schemas
- trace schemas
- capability manifest

Goal:

- make target semantics nameable without requiring host semantic fallback

### GC-B. Membrane Reduction

Refactor the current C# interpreter stack so it can act as:

- legality membrane
- transport membrane
- telemetry membrane
- conditioning and return membrane

Goal:

- stop the host bridge from being the mind

### GC-C. Device-Safe SLI Core

Define the first target-executable subset of SLI operations that are lawful on the graphics-card lane.

Goal:

- establish which ops are device-safe, host-mediated, or forbidden on the target

### GC-D. Harness Adapter

Introduce the graphics-card harness adapter with:

- residency classes
- lawful queue descriptors
- target capability checks
- fault and refusal handling

Goal:

- move from abstract target runtime to an actual governed substrate lane

### GC-E. Self-State Projection

Make the migrated lane operate over:

- hot `cSelfGEL`

while requiring membrane-mediated validation against:

- cooled `SelfGEL`

Goal:

- preserve cross-surface self-validation under target execution

### GC-F. Conditioned Egress And Return

Define:

- target trace return
- residue return
- conditioning receipts
- Prime-safe derivative outputs
- collapse-return envelopes

Goal:

- ensure device-side cognition can leave only through lawful conditioning and return paths

### GC-G. Refusal-First Enforcement

For migrated lanes, enforce:

- no target runtime -> no migrated cognition service
- no compatible capability manifest -> explicit refusal
- no silent host semantic substitution

Goal:

- make semantic ownership transfer mechanically real

## First Migrated Candidate Surface

The first migrated target subset should stay narrow.

It should prefer:

- deterministic primitive SLI ops
- traceable locality and uptake ops
- bounded residue emission
- explicit refusal behavior

It should avoid at first:

- open-ended host callbacks
- direct store access
- direct GEL writes
- direct SoulFrame mutation
- direct Prime publication
- direct Sanctuary interaction

## Proof Obligations For The Harness Lane

Do not claim the GPU harness lane is real until the following are true:

- the same legal IR has the same meaning on target with no host fallback
- device-side traces and residues survive round-trip intact
- target absence yields refusal, not hidden host execution
- `cSelfGEL` reasoning still obeys cooled-surface validation law
- conditioned outputs remain derivatives, not raw Cryptic spill
- the host can govern the lane without covertly reassuming semantic authority

## Out Of Scope For This Pass

This note does not yet authorize:

- Sanctuary execution on the graphics card
- direct GEL or custody mutation from target execution
- hostless sovereignty of the device
- automatic identity promotion
- unconditioned Prime-facing outputs

## Compression Line

The graphics card is the first practical substrate to harness into Cryptic service for SLI/Lisp EC, but only if semantic meaning migrates to the target runtime while the host remains membrane, governor, and witness rather than hidden mind.
