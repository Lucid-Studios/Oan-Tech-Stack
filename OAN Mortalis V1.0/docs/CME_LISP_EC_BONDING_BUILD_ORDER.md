# CME/Lisp EC Bonding Build Order

## Purpose

This build note stages the first enacted higher-order locality slice for CME-bound Lisp/SLI execution context work.

The implementation ladder is:

`Bond -> Perspective -> Participation -> Rehearsal -> Witness -> Transport`

`A1` and `B1` implement only the first three runtime-local thresholds:

- Bonded locality substrate
- Perspectival configuration
- Participatory configuration

Dream/Game rehearsal, witness formation, morphism candidacy, lawful transport, admissibility, and Sanctuary-governed accountability remain deferred.

## Ownership

`CradleTek hosts. SoulFrame governs. AgentiCore inhabits. SLI/Lisp forms higher-order cognitive locality before it ever transports it.`

This slice keeps the ownership split explicit:

- C# hosts, validates, materializes, snapshots, and enforces order
- Lisp composes bounded local structure

## Naming

The architecture note uses `bonding` because the implementation sequence begins with a bonded locality substrate.

The runtime code uses `locality` and `anchor` names instead of raw `bond-*` operator names so the new higher-order locality surface does not collide with the repo's existing bonded-authority and governance language.

## Scope

### A1 foundation

- add an internal higher-order locality state lane to `SLI.Engine`
- add typed residue classes from day one
- add `locality.lisp` with bounded primitive and composite locality forms
- add a deterministic composition expander that loads only the locality composites from `locality.lisp`

### B1 execution

- materialize locality, perspective, and participation state through primitive symbol handlers
- execute Lisp-authored composite forms through bounded expansion
- insert locality, perspective, and participation into the canonical cognition cycle before `compass-update`
- prove differential formation in tests using the same C# fixture context with different Lisp-authored locality programs

## Deferred

This slice does not implement:

- Dream/Game rehearsal
- witness packets
- morphism candidacy
- lawful transport
- admissibility review
- Sanctuary accountability intake
- governance mutation
- custody mutation
- identity binding

## Runtime Boundary

The first slice proves enacted locality only.

It must remain:

- bounded
- non-custodial
- non-governing
- non-transporting
- pre-admissible
- pre-accountability
