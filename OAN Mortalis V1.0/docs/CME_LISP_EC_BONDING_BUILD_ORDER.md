# CME/Lisp EC Bonding Build Order

## Purpose

This build note stages the first enacted higher-order locality slice for CME-bound Lisp/SLI execution context work.

The implementation ladder is:

`Bond -> Perspective -> Participation -> Rehearsal -> Witness -> Transport`

`A1` and `B1` implement only the first three runtime-local thresholds:

- Bonded locality substrate
- Perspectival configuration
- Participatory configuration

The next bounded step is `C`, which implements Dream/Game rehearsal as an explicitly invoked exploratory lane rather than as a required canonical cognition step.

The current maturity sequence is:

- `A1+B1 = lawful inhabitation`
- `C = lawful rehearsal`
- `D = witness / morphism candidacy`
- `E = lawful transport`

Dream/Game rehearsal, witness formation, morphism candidacy, lawful transport, admissibility, and Sanctuary-governed accountability remain deferred from the canonical default lane.

## Ownership

`CradleTek hosts. SoulFrame governs. AgentiCore inhabits. SLI/Lisp forms higher-order cognitive locality before it ever transports it.`

This slice keeps the ownership split explicit:

- C# hosts, validates, materializes, snapshots, and enforces order
- Lisp composes bounded local structure

## Naming

The architecture note uses `bonding` because the implementation sequence begins with a bonded locality substrate.

The runtime code uses `locality` and `anchor` names instead of raw `bond-*` operator names so the new higher-order locality surface does not collide with the repo's existing bonded-authority and governance language.

### C rehearsal

- add an internal rehearsal state lane nested under higher-order locality

- add an internal rehearsal execution path that remains non-binding, pre-admissible, non-transporting, and non-canonical by default

### D witness / morphism candidacy

- add an internal witness state lane nested under higher-order locality
- add bounded witness primitives and witness composite expansion in Lisp
- add an internal witness execution path that compares lawful localities and rehearsal branches without granting transport

## Runtime Boundary

The first slice proves enacted locality only.

It must remain:

- bounded
- non-custodial
- non-governing
- non-transporting
- pre-admissible
- pre-accountability

Rehearsal extends that boundary without changing its law:

- rehearsal is internal-first
- rehearsal is explicitly invoked
- rehearsal is not a required canonical cognition step
- rehearsal may vary a world, but it may not vary who is lawfully bound to that world

Witness extends the boundary without crossing it:

- witness is internal-first
- witness compares lawful localities and rehearsal branches before any gluing claim
- witness may name preserved structure, lawful difference, and residue
- a candidate morphism is not yet a transport permission
