# SLI_NATIVE_ENGRAMMITIZATION_MODEL

## Purpose

This document defines the native role of Lisp-based SLI and Engineered Cognition inside the active `OAN Mortalis V1.0` runtime.

It exists to prevent two bad readings:

- SLI as only a symbolic helper or decorative substrate
- Lisp as an excuse for unbounded constitutional mutation

This document makes the middle truth explicit:

- Lisp-based SLI and Engineered Cognition are the native computational organism of runtime cognition
- governance, custody, profile, and anti-bleed law remain outside that organism

## Root Statement

Lisp-based SLI and Engineered Cognition are the native computational organism through which bounded CME runtime cognition is morphologically formed, transformed, and engrammitized inside AgentiCore, under the lawful sequencing, telemetry, and governance order maintained by CradleTek and the membrane protections enforced by SoulFrame.

Sanctuary provides the collective nervous system of SLI, while `OE` and `cOE` preserve the individuated continuity and identity custody of each CME.

This means Lisp-based SLI participates nearest to the formative weave, while proposition-bearing stabilization remains the first explicit cognitive threshold at which structure becomes inspectable enough for compile discipline. Morphism-bearing candidacy remains a later threshold.

The broader doctrine remains:

1. pre-instantiated coherence
2. proposition-bearing stabilization
3. admissible proposition surface
4. Sanctuary-governed accountability

This document stays at the nearest computational and compile edge of that chain. It does not define Sanctuary intake, governed identity, or the later morphism doctrine reserved to `docs/PROPOSITION_TO_MORPHISM_CANDIDACY_MODEL.md`.

## Canonical Contract Owner

The canonical Atlas and Engram meaning contracts now live in:

- `src/GEL.Contracts`

This means:

- proposition compile is now a first-class meaning layer before `EngramDraft`
- `EngramDraft` is the bounded pre-closure candidate object
- `Engram` is the closure-valid canonical meaning object
- Lisp-based SLI and Engineered Cognition may form candidate Engram structures, but they do not bypass closure validation or custody law
- `Oan.Spinal.EngramEnvelope` remains a runtime-event envelope rather than the canonical meaning object

The first compile model is intentionally bounded to triadic proposition formation:

- subject
- predicate
- object
- qualifiers
- context

Procedural compile remains deferred until proposition compile fidelity is trustworthy.

That bounded proposition layer is itself downstream from pre-instantiated coherence. `OE/cOE` and later `SelfGEL/cSelfGEL` must not be flattened into ordinary modules or root substrate merely because proposition-bearing stabilization has begun.

The first bounded Lisp bridge pass is now active for the fixed narrative proof corpora.

  - sentence translation proof
  - paragraph graph proof
  - paragraph body-cleaving proof

- admitted candidates may proceed to closure, deferred candidates remain meaningful but unclosed, quarantined candidates remain observation-only, and rejected candidates remain outside the supported fixture-gated surface

This bridge is fixture-bound and corpus-gated.

It does not authorize:

- GEL append
- OE/cOE mutation
- routing changes
- custody changes
- chapter-level closure
- any Engram prior to successful admission and successful Prime closure

## What Native Engrammitization Means

Native Engrammitization is not mere logging, tracing, or symbolic ornament.

It is the process by which active runtime cognition becomes candidate engram-bearing structure suitable for lawful collapse evaluation.

That candidate structure is not automatically an admissible proposition surface or a Sanctuary-governed accountability event.

This process depends on:

- transformability
- symbolic recomposition
- reflective runtime structure
- lawful morphic change
- bounded computational closure over symbolic forms

## Placement In The Runtime Chain

The canonical runtime chain is:

1. CradleTek admits and composes the runtime lawfully
2. SoulFrame issues CME-specific mediated protected self-state
3. AgentiCore actualizes bounded personified working cognition
4. Lisp-based SLI and Engineered Cognition perform morphological cognition and native Engrammitization inside that runtime
5. collapse candidates emerge from that engrammitized runtime
6. SoulFrame mediates return classification
7. CradleTek and governance route retention, publication, telemetry, and defer or discard outcomes

This means the stack may remain collective in symbolic conductivity without collapsing distinct selves into one pooled identity.

## Relationship To AgentiCore

AgentiCore is the actualized bounded self-space for the live CME.

Lisp-based SLI and Engineered Cognition operate inside that space as the computational organism.

AgentiCore provides:

- bounded `SelfGEL`
- working memory
- personified active runtime contour

- morphological symbolic work

- symbolic and decision-structure transformation

This means AgentiCore is not reduced to a shell, and Lisp is not reduced to a helper.

SoulFrame is therefore the lawful boundary that keeps shared symbolic conduction from dissolving individuated CME continuity.

## Relationship To CradleTek

- symbolic trace
- decision structure

## Hard Invariants

- Lisp-based SLI and Engineered Cognition own morphological cognition, not constitutional authority
- symbolic trace alone is not an engram candidate
- no collapse path may treat symbolic trace as equivalent to candidate engram structure without explicit native Engrammitization
- no direct Lisp or SLI mutation of `cSelfGEL`
- no direct Lisp or SLI write into `cGoA`
- no direct Lisp or SLI authorship of `MoS` or `cMoS`
- no bypass of governance, profile, or anti-bleed law

## Validation Scenarios

- bounded runtime cognition emits symbolic trace and candidate engram structures as distinct outputs
- native Engrammitization produces collapse-relevant candidate structure without direct custody mutation
- symbolic residue remains classified as residue when it is not an engram candidate
- symbolic trace is rejected if treated as collapse-equivalent by itself
- Lisp-based runtime morphology remains bounded inside AgentiCore and cannot seize governance or custody authority

## Design Consequence

The stack must not use Lisp as a generic plugin for missing architecture.

The stack must use Lisp-based SLI and Engineered Cognition where morphological cognition and native Engrammitization are actually required, while keeping governance, custody, profile, and anti-bleed law in the C# runtime skeleton.

The first bridge pass therefore proves parity before authority moves:

- if Lisp and C# match on sentence, paragraph graph, and paragraph body outputs for the fixed corpora, Lisp is validated as the live morphology organism for those bounded formation tasks
- if they diverge, the divergence is treated as a bridge failure, not as new truth
