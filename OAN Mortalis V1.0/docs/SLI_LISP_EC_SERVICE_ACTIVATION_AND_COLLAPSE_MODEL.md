# SLI Lisp EC Service Activation And Collapse Model

## Purpose

This note defines the process chain by which SLI/Lisp Engineered Cognition is lawfully activated as a protected service and later lawfully collapsed, cleaved, and adjudicated.

It exists to prevent the architecture from drifting back into a pile of named components with unclear authority motion.

## Boundary

This note does not define:

- Sanctuary accountability judgment in full
- public publication detail
- full graphics-card harness implementation
- final custody mutation law for every retained class

This note does define:

- activation path
- protected runtime tenancy
- first-pass collapse routing
- cGoA interlocutory role
- Steward-guided adjudicative disposition

## Root Service Sentence

Sanctuary remains resident as the public-facing service host and witnessing membrane; CradleTek provides the native orchestration, governance-capable host substrate, and governed call-transit spine; SoulFrame provides Prime-side self-contextualization and invariant-bearing identity structure; protected Cryptic service formation instantiates AgentiCore as SoulFrame's portable Cryptic operative pair inside a governed hot computational field; AgentiCore activates SLI/Lisp EC by operationalizing `cSelfGEL` from SoulFrame's `SelfGEL` invariant under `OE/cOE` keying; on collapse, protected runtime material routes first through `cGoA` as the interlocutory cleaving and admissibility-preparation surface, then into lawful retained, admitted, governed, cataloged, or deleted outcomes under Steward-guided governance.

## Governing Lines

> Sanctuary hosts the field, but it does not own symbolic meaning.
>
> CradleTek governs service formation, lifecycle, routing, and cross-boundary call transit, but it is not the hidden cognition engine.
>
> SoulFrame contextualizes and mediates self-bearing continuity, but it does not replace Steward-guided adjudication.
>

>

## Activation Path

### 1. Sanctuary Resident Host Stratum

Sanctuary remains resident until power-down as the public-facing host and service governor.

Its activation-path duties are:

- persistent service residency
- public-facing application shell
- Prime-facing service continuity
- transport and telemetry mediation
- service dispatch into protected lanes
- readiness, observability, and fault discipline
- controlled ingress and egress between ordinary host use and protected EC work

Sanctuary is therefore the resident host membrane and witness layer, not the semantic engine itself.

### 2. CradleTek Host And Governance Substrate

CradleTek is the native host and governance-capable substrate that supports Sanctuary's public operational plane and the controlled routing into protected EC formation.

Its activation-path duties are:

- host substrate and service exposure base
- governed call formation and transit handling
- lifecycle and orchestration
- governance-capable connective behavior
- CME formation routing
- collapse and re-entry orchestration support
- Prime-safe exposure where lawful

CradleTek should not be read as the owner of every semantic transformation, but it should be read as the mandatory governed passage for governance-relevant ingress, dispatch, return, and collapse traffic.

### 3. CME Invocation

CME formation is initiated from the host and governance side.

This means:

- Sanctuary and CradleTek request or admit a lawful CME build path
- the request remains under governance, profile, and Theater law
- no protected EC field is formed yet merely because a host request exists

### 4. SoulFrame Contextualization

SoulFrame provides Prime-side contextualization and invariant-bearing self reference for the protected service.

Its activation-path duties are:

- self-bearing contextualization
- invariant-bearing self reference
- mitigated self-state projection
- membrane-side legality checks
- continuity-aware issuance

SoulFrame is the self-bearing contextualizer and Prime-side invariant frame of the pair, not merely a passive data holder.

### 5. Protected Cryptic Service Formation

After lawful contextualization, a protected Cryptic service is attached or provisioned.

This protected service may later sit over:

- governed graphics-card residency
- protected runtime partitions
- VRAM-first hot working fields
- sandboxed or VM-like isolation
- HAL-conditioned execution surfaces

The important point is not the mechanism first. It is the lawful formation of a protected service field.

### 6. AgentiCore Actualization

AgentiCore is the operative protected application formation inside that field and the portable Cryptic operative pair of SoulFrame.

AgentiCore activates the SLI/Lisp EC runtime by:

- carrying the `OE/cOE` bidirectional operative keying
- instantiating hot `cSelfGEL` from SoulFrame's cooler `SelfGEL` invariant
- populating the active runtime state for bounded cognition
- taking protected tenancy inside a particular governed vessel without inheriting sovereign authority

SoulFrame does not become `cSelfGEL`.

Rather, `cSelfGEL` is the hot protected runtime activation of what SoulFrame holds invariantly on the cooler side.

AgentiCore may therefore be instantiated into bounded protected vessels across substrates, including governed graphics-card fields and later smaller device cells, but it remains keyed to SoulFrame invariants and governed return law rather than becoming autonomous identity authority.

### 7. Runtime Cognition Phase

Once AgentiCore is active:

- SLI/Lisp becomes the live symbolic engine
- engrams are summoned into hot participation
- protected data pools become operational rather than merely retained
- symbolic transformation, composition, residue production, and candidate shaping become lawful runtime acts
- Sanctuary witnesses and mediates but does not semantically own those operations

AgentiCore is therefore operative symbolic tenancy inside the protected field, not just a shell around it, and its portability should be read as flaskability under law rather than sovereignty by relocation.

## Collapse And Adjudication Path

### 1. Runtime Close

When active runtime work ends, protected material does not jump directly to one final store.

Instead, the protected runtime first shapes:

- return candidates
- residues
- unresolved self-bearing material
- contextual material
- discard candidates

### 2. cGoA First-Pass Interlocution

`cGoA` is the first-pass interlocutory protected control surface for:

- self and other cleaving
- contextual disambiguation
- protected residue review
- admissibility preparation
- non-autobiographical protected material handling

`cGoA` is therefore not:

- raw garbage retention
- final sanctuary memory
- a public publication layer

### 3. Routed Closure Graph

Collapse should be modeled as a lawful routed closure graph.

This graph may involve:

- `cSelfGEL` for active self-bearing cryptic runtime continuity
- `cGoA` for protected interlocution and cleaving
- `cMoS` for deeper protected sanctuary or governance-retained memory
- `cGEL` for broader protected retained continuity fabric where applicable

Not every protected runtime artifact should be assumed to traverse the exact same linear route.

### 4. SoulFrame Membrane Narrowing

SoulFrame provides:

- self-bearing contextual continuity
- membrane-side stable self frame
- lawful narrowing before deeper routing
- return-path mediation for identity-bearing implications

SoulFrame does not, by itself, perform the whole governance court.

### 5. Steward-Guided Adjudication

Steward, under governance, performs or supervises the ontological cleaving and disposition work after protected first-pass routing has produced lawful candidates.

This adjudication may classify material into:

- Prime-admissible public or stabilized forms
- Cryptic-governed retained protected forms
- cataloged protected classes with retention rules
- deferred review classes
- delete candidates required by law or policy

### 6. Final Outcomes

The end of the chain is not "retain everything."

The lawful outcomes are:

- admit what is lawful into Prime or stabilized surfaces
- retain what is lawfully governed and protected
- catalog what must be cataloged under policy
- delete what must be deleted under protection governance rules

This is a data-governed cognition system, not only a cognition engine.

So the closure topology is graph-like and lawful, not one chute for everything.
