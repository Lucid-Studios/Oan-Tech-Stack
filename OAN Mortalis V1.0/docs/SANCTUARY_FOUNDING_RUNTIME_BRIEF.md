## 0. Purpose of this brief

This brief guides the next implementation phase of the OAN Tech Stack.

It is written for a project-scoped GPT to refine and stabilize before it is passed to Codex.

The implementation target is:

> Build the internal governance runtime first, before any operator-bound CME stacks are introduced.

This means the first lawful active CMEs in the system are the internal governance CMEs:

- Mother of Prime
- Father of Cryptic
- Steward of Sanctuary

The purpose of this phase is to ensure Sanctuary can:

- be lawfully founded
- be lawfully re-entered
- maintain runtime continuity
- preserve Prime/Cryptic separation
- operate with a triangulated integrity model
- support future operator-bound CME stacks without structural compromise

## 3. Core terms and canonical meaning

### 3.1 Theater

A Theater is a locally bounded cognitive universe with:

- its own geometry
- its own symbolic grammar
- its own admissible operations
- its own lawful transitions

Primary theaters:

- Prime Theater - public expression, discourse, publication
- Cryptic Theater - sealed identity, security, certification, protected continuity
- Dream Theater - speculative, generative, exploratory cognition

Theaters must not be collapsed for convenience.

### 3.2 OE (Opal Engram)

OE is the identity spine and witness chain.

It stores:

- commitments
- chain state
- references to lawful stores
- founding and bonding references
- profile references
- role continuity references
- headers and endcaps later as needed

OE must not become a large payload store.

### 3.3 SelfGEL, cSelfGEL, GoA, cGoA

These are lawful storage domains.

- SelfGEL - Prime-safe self-continuity and autobiographical artifacts
- cSelfGEL - Cryptic self-continuity and sealed autobiographical and certification artifacts
- GoA - world/context memory and operational environment artifacts
- cGoA - sealed or restricted contextual and environmental artifacts

OE points to them. It does not absorb them.

### 3.4 Internal governance CMEs

The first lawful active CMEs are:

- Mother of Prime
- Father of Cryptic
- Steward of Sanctuary

They are not generic agents. They are standing constitutional offices.

### 3.5 Operator profile

An Operator Profile is a constitutional environment selector.

Current v0.1 supported tiers:

- `personal`
- `enterprise`
- `government`

These are hard override profiles.

### 3.6 Seal admission

Trusted symbolic participation must depend on lawful state and charter alignment, not only on emitted data.

Telemetry is evidence, not root trust.

## 8. Runtime state and telemetry doctrine

The system must treat runtime integrity as triangulated.

### 8.1 Triangulated state inputs

Each of Mother, Father, and Steward must emit:

- `state`
- `confidence`
- `reason_codes[]`
- `observed_anomalies[]`
- `policy_status`
- `lease_status`

State domain:

- `GREEN`
- `YELLOW`
- `RED`
- `LOCK`

### 8.2 State resolution rules

Minimum v0.1 resolution:

- any `LOCK` -> runtime `LOCK`
- two or more `RED` -> runtime `RED`
- one `RED` with the others `GREEN` or `YELLOW` -> runtime `YELLOW` or `RED` depending on failure axis
- any `YELLOW` with no `RED` or `LOCK` -> runtime `YELLOW`
- all `GREEN` -> runtime `GREEN`

Required outputs:

- `runtime_state`
- `primary_failure_axis`
- `operator_attention_required`
- `escalation_required`
- `checkpoint_recommended`

For v0.1, continuity failures weighted through Steward should be stricter than expression-only failures weighted through Mother.

### 8.3 Telemetry namespaces

Extend cognition telemetry with:

- `metrics.layer0.operator`
- `metrics.layer0.founding`
- `metrics.layer0.reentry`
- `metrics.layer0.governance`
- `metrics.layer0.triangulated_state`

### 8.4 Denial reason code namespaces

All denial reason codes must be namespaced:

- `FOUNDING_*`
