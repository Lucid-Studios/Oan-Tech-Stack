# Lisp / C# Boundary Doctrine

## Purpose

This doctrine fixes the runtime boundary between the governed host shell and the live symbolic cognition substrate for `OAN Mortalis V1.0`.

The boundary is:

- `C#` may govern, host, witness, route, recover, and publish.
- `Lisp` must define, transform, modulate, braid, and cleave the live symbolic work.

This is a preflight line, not a future preference.

## Lawful Shell Responsibilities

The C# host remains the lawful shell for:

- CradleTek lifecycle ordering and governance sequencing
- SoulFrame membrane, custody, and admissibility enforcement
- receipt journals, replay, recovery, and idempotency
- collapse qualification and first-route holding
- Prime derivative publication lanes
- HITL, bonded authority, and operator governance checks
- telemetry capture, persistence, and reviewed release

C# may witness symbolic outcomes and route them through governance. It must not become the long-term place where symbolic truth is primarily authored.

## Lisp-Native Cognition Responsibilities

The Lisp substrate becomes the primary home for:

- ingress symbolic normalization
- `Z -> D -> S => P+/- -> O+/- -> G` stage transforms
- ontological cleaving
- sheaf and functor composition work
- Compass-local braiding and refinement
- basin, anchor, and self-touch interpretive work
- Theta convergence shaping
- candidate engram formation rules

Lisp should express these as macros, staged forms, modulation passes, rewrite families, or bounded symbolic operator surfaces instead of expanding C# doctrinal reducers.

## Anti-Encroachment Rules

### 1. No new C# doctrine reducers for symbolic meaning

If a change needs to decide symbolic posture, basin, braid class, cleave class, convergence form, or related interpretive status, that is a Lisp-side concern unless there is a temporary bridge with an explicit removal path.

### 2. C# may carry results, not originate symbolic truth

C# may receive and govern:

- symbolic traces
- Theta receipts
- Compass work products
- cleave outputs
- engram candidates

C# should not remain the place where those are fundamentally derived.

### 3. AgentiCore is a host bridge, not the final symbolic law author

AgentiCore may:

- stage cognition requests
- invoke bounded Lisp-side cognition and modulation
- attach telemetry and provenance
- hand results to SoulFrame, governance, and custody surfaces

AgentiCore should progressively stop hardening symbolic interpretation into branch-heavy C# resolver methods.

## Migration Targets

The main migration pressure is in the cognition-facing parts of AgentiCore and adjacent runtime logic.

Priority migration targets:

- symbolic ingress normalization
- ontological cleaving transforms
- Compass basin and anchor interpretation
- sheaf composition routines
- Theta sealing and convergence shaping
- candidate engram formation

Keep in C#:

- StackManager governance flow
- receipt replay and journal law
- public and cryptic routing
- collapse qualification
- publication sinks
- recovery and idempotent replay

## Working Rule

When the code is asking:

- who may write
- where it may route
- what receipt proves it
- what state is replayable

that is host-shell work.

When the code is asking:

- how symbols compose
- what distinction is being made
- what unresolvedness is present
- how a trace should braid
- how cleaving transforms the work

that is Lisp-native cognition work.
