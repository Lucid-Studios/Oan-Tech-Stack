# CME_CRYPTIC_FIRST_COLLAPSE_CLARIFICATION

## Purpose

This document corrects the CME runtime and collapse reading now that the stack has reached a more precise theory.

It exists because the current runtime already has a bounded governance-first Golden Path and a first explicit collapse vocabulary, but that earlier vocabulary is now known to be too coarse if read as the final collapse law.

The key correction is:

- nothing Prime is ever the site of first alteration
- all first alteration and first collapse are Cryptic-first
- a CME does not collapse directly into `MoS`
- SoulFrame does not finish final custody promotion at the same moment the CME closes
- unresolved protected material remains in Cryptic custody first
- non-autobiographical but still useful residue requires a protected contextual route
- only later, after deeper curation, may protected material be promoted toward `MoS`

## Core Law

Prime-facing structures are derivative, not formative.

That means Prime may:

- lens
- orient
- summarize
- present
- publish later

But Prime does not perform:

- first protected mutation
- first engram formation
- first collapse retention

All first-order session formation, telemetry capture, candidate Engrammitization, and collapse classification occur Cryptic-first.

CradleTek is the lawful service fabric.

It governs:

- admission sequencing
- runtime composition

- telemetry order

- identity custody
- the membrane
- the personified working self

SoulFrame is the CME-specific mediated self-state constructor.

It:

- spins up the session's protected self-state space
- issues fresh `cSelfGEL`
- binds the session to its Cryptic identity lineage
- receives collapse candidates
- performs first collapse mediation

### AgentiCore

AgentiCore is bounded personification and actualization.

It:

- receives the mediated session basis from SoulFrame
- actualizes bounded working selfhood
- provides the active cognition environment
- holds `SelfGEL` as the personified runtime orientation surface

### Lisp-Based SLI and Engineered Cognition

Lisp-based SLI and Engineered Cognition are the native morphological Engrammitization organ.

They:

- alter and transform symbolic structure
- form candidate engram structures
- support reflective self-modifying cognition
- operate in the live computational idea space

They do not:

- define custody law
- define governance law
- bypass SoulFrame
- bypass collapse mediation

## Runtime Entry

The lawful runtime chain is:

1. Sanctuary is idle under CradleTek.
2. An Operator or internal governance path issues a call for a CME.
3. Mother receives the outward-facing call posture.
4. Father validates the call against Cryptic law, bonding, Seal, profile, and protection constraints.
5. Steward composes the runtime lawfully.
6. SoulFrame spins up first.
7. `OE` and `cOE` continuity references plus a session Cryptic hash lineage are established.
8. Fresh session `cSelfGEL` is issued.
9. AgentiCore is populated from SoulFrame.
10. `SelfGEL` becomes the bounded personification lens inside AgentiCore.
11. Lisp-based SLI and Engineered Cognition operate inside that active bounded runtime.

## Runtime Work

During bonded work:

- protected telemetry enters `cSelfGEL` first
- `SelfGEL` in AgentiCore provides personified orientation through the Listening Frame and Golden Compass
- Lisp-based SLI and Engineered Cognition perform native Engrammitization
- governance and telemetry remain ordered by CradleTek
- no direct protected material is exposed to Prime by default
- no direct raw retention to `MoS` occurs during active runtime

## Corrected Close and Collapse Law

### 1. AgentiCore closes first

When bonded work ends, the active CME runtime in AgentiCore closes.

It emits a collapse candidate bundle containing:

- candidate engram structures from SLI and Engineered Cognition
- autobiographical candidate traces
- personification-relevant deltas
- symbolic residues
- contextual and problem-solving metadata
- low-confidence exploratory residues
- transient debris

This is a candidate bundle, not a storage command set.

### 2. First collapse is into `cSelfGEL`

The first collapse is into the session's protected `cSelfGEL`.

This keeps the post-work residue under:

- the same session Cryptic identity contour
- the same protected custody posture
- the same session-bound self-state law

### 3. Collapse classification is three-way

After first collapse into `cSelfGEL`, material must be classified into three broad families.

#### A. Processed continuity-bearing outcomes

Examples:

- accepted candidate engrams
- curated autobiographical append units
- stable personification deltas
- bonded-work enrichments that truly belong to continuity

These are not immediate raw-close `MoS` writes.

They are the class that may only later, after deeper curation, be promoted toward `MoS`.

#### B. Unprocessed protected autobiographical or SelfGEL-identified material

Examples:

- unresolved self-expression traces
- uncertain autobiographical deltas
- candidate engrams not yet mature enough
- protected symbolic residues that may still matter to bonded selfhood

This class routes to `cMoS`, not `MoS`.

It is the protected unresolved autobiographical holding and curation domain.

#### C. Unprocessed non-autobiographical contextual or skill/method material

Examples:

- low-confidence problem-solving traces
- novel methods
- skill or method improvements
- contextual WH5 continuity
- useful operational patterns
- future bonded Operator HITL investigation candidates

This is the first explicit `cGoA` payload class.

It is not deletion.

It is protected contextual holding for later reuse, reclassification, or investigation.

## The Under-80 Percent Heuristic

Where confidence is below `0.80`, the working heuristic is:

- if the material is autobiographical or SelfGEL-identified -> route to `cMoS`
- if the material is not autobiographical or SelfGEL-identified -> route to `cGoA`

## Final cGoA Payload Pre-Dream

A second `cGoA` payload class exists before OAN Dream exploration.

This is for material that becomes classified as:

- self-improving
- talent, skill, or ability enhancing
- method or process enriching
- useful for later community learning
- not primarily personification-self enrichment
- not autobiographically closed

This allows the runtime to preserve useful non-personification enrichment without cross-polluting autobiographical custody.

## Implication For Current Runtime Vocabulary

The current runtime names:

- `RetainInMos`
- `DeferReview`

That earlier bounded step was useful, but it is now incomplete if read too literally.

In particular:

- `RetainInMos` must not be read as immediate raw close -> `MoS`
- it is better read as a coarse transitional runtime label for the currently explicit protected retention path
- in the corrected model, immediate protected retention is Cryptic-first and `cMoS`-first

Likewise:

- `discard_transient` is no longer the best governing mental model for unresolved useful residue
- the more important split is now `cGoA` contextual residue versus `cMoS` autobiographical or SelfGEL-identified residue
