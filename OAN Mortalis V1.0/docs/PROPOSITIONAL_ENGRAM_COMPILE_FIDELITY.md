# PROPOSITIONAL_ENGRAM_COMPILE_FIDELITY

## Purpose

This document defines the first proposition-first compile layer for the active `OAN Mortalis V1.0` stack.

It exists to make one rule explicit:

- bad propositions -> bad procedures -> bad cognition

The stack must therefore prove lawful propositional compile fidelity before it is trusted with procedural compile fidelity or deeper engineered cognition patterns.

## Canonical Compile Chain

The bounded first cognitive chain is:

```text
ingress recognition
-> proposition stabilization
-> candidate proposition engram
-> membrane submission
-> Prime closure
```

This chain makes the order explicit:

- ingress is not yet thought
- proposition stabilization is the first act of thought
- closure comes after meaning, not before it

This chain presupposes a prior coherence regime. The proposition layer begins only after pre-instantiated coherence has yielded a stable enough surface for repeatable proposition-bearing formation.

Within the broader doctrine defined in `docs/WEAVE_TO_PROPOSITION_STABILIZATION_MODEL.md`, this compile layer occupies only the second threshold:

1. pre-instantiated coherence
2. proposition-bearing stabilization
3. admissible proposition surface
4. Sanctuary-governed accountability

## Bounded First Model

The first proposition compile pass uses a bounded triadic proposition shape only:

- subject
- predicate
- object
- qualifiers
- context

Procedural compile is explicitly deferred.

Morphism-bearing stabilization is also explicitly deferred. This pass does not treat proposition-bearing stabilization as though it already implied lawful transport or cross-frame capability.

This means the first compile layer is intentionally narrow. It is intended to prove proposition fidelity, not procedural pathing.

## First Proof Target

The first bounded proposition-fidelity target is:

- the hybrid protected-ingress rehearsal surface

This target is chosen because it pressures:

- referents
- authority relations
- governance posture
- reveal constraints
- masked symbolic handles

without requiring broad narrative ambiguity or procedural expansion.

## Fidelity Criteria

The proposition compile layer is judged by:

- referent fidelity
- predicate fidelity
- modifier fidelity
- scope and context fidelity
- closure fitness

The proposition layer therefore sits between formative coherence and later morphism candidacy. It is the first inspectable cognitive threshold, not the whole of admissibility or transport law.

Admissibility review is downstream from proposition stabilization, and Sanctuary accountability is downstream from admissibility.

### Referent Fidelity

- subject and object identity roles remain correct
- masked symbolic handles remain attached to the correct proposition terms

### Predicate Fidelity

- the asserted relation remains correct
- generic proposition roots remain stable and atlas-resolved

### Modifier Fidelity

- authority role and reveal-related qualifiers remain explicit
- policy-relevant qualifiers are not silently dropped

### Scope and Context Fidelity

- boot class
- expansion posture
- reveal posture
- protected-ingress class

remain visible as proposition context rather than disappearing into raw parser residue.

### Closure Fitness

- only stable proposition candidates may project into `EngramDraft`
- the projection must remain lawful under the existing membrane and Prime closure validator

## Invariant

Generated proposition candidate does not equal accepted truth.

Latently formed structure is therefore not automatically a proposition, and a proposition-bearing candidate is not automatically a governed identity.

The first compile pass therefore preserves these invariants:

- generated structure != accepted truth
- formed candidate != governed identity
- latent weave != accountable surface

The first compile pass therefore distinguishes:

- proposition candidate
- projected `EngramDraft`
- accepted closed Engram

Lisp may form proposition candidates.

C# remains the oracle for proposition parity, the materializer of canonical proposition contracts, the membrane submitter, and the closure authority.
