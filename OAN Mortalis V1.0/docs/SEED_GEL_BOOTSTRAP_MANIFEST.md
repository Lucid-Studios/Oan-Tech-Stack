# SEED_GEL_BOOTSTRAP_MANIFEST

## Purpose

This document defines what the first seed `GEL` must load canonically before broader symbolic contextualization generation is allowed.

It is a bootstrap manifest and doctrine note, not a runtime append implementation.

## Genesis Contents

The first seed `GEL` should load only the following canonical classes:

1. first atlas roots
2. first grammar operators
3. first reserved disciplinary domains
4. first governance/meta symbol domains
5. first constructor legality tables
6. first known/unknown extension marker set
7. first bridge policy rules

## Canonical At First Boot

Canonical at first boot means:

- pinned `RootAtlas`
- curated seed lemma subset pinned under the canonical root surface
- lemma-root identity fixed for canonical root keys
- pinned symbolic domain constitution
- deterministic constructor-body law
- explicit closure grades
- explicit extension-state semantics

## Not Canonical At First Boot

The following remain out of scope for first seed genesis:

- full symbolic contextualization generation
- Dream-bound symbolic movement
- public/shared memory mutation
- broad disciplinary overlay synthesis
- heuristic domain bridging

## Seed Loading Order

The intended bootstrap order is:

1. load symbolic domain constitution
2. load canonical `RootAtlas`
3. load curated seed lemma roots
4. validate constructor legality against reserved domains
5. prove English intake lands on admitted lemma roots
6. admit `EngramDraft` binding against canonical roots
7. validate closure grade
8. only later consider governed memory movement

## Immediate Consequence

The first seed `GEL` must begin life with law, not just content.

That means:

- the symbolic world is reserved before it is extended
- root cores are explicit before merged forms are generated
- the first seed basis is curated before broad Engram generation begins
- unknown space is articulated before the system pretends completeness
