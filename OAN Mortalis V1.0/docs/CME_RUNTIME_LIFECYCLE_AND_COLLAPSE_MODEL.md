# CME_RUNTIME_LIFECYCLE_AND_COLLAPSE_MODEL

## Purpose

This document defines the lawful runtime lifecycle of a CME in the active `OAN Mortalis V1.0` stack.

It exists to answer four questions before further module-by-module growth:

- what a CME runtime object is
- where its state lives at each phase
- what may move between phases
- what must collapse where on close

This document extends the already-implemented governance-first runtime. It does not replace:

- the Golden Path
- the runtime hardening layer
- the control plane
- the current SoulFrame membrane contracts
- the bounded AgentiCore worker pattern

## Root Runtime Sentence

A CME instantiates from lawful mediated continuity, thinks in bounded `SelfGEL` inside AgentiCore, uses Lisp-based SLI and Engineered Cognition as the native morphological and engrammitizing organism during runtime, and collapses Cryptic-first by mediated transfer from `SelfGEL` into `cSelfGEL`, then first into `cGoA` or `cMoS` according to protected classification, with any later `MoS` promotion or Dream eligibility occurring only after deeper curation.

## Runtime Roles

### CradleTek

CradleTek is the lawful service, governance-sequencing, telemetry, and orchestration fabric.

It owns:

- runtime composition
- service ordering
- governance sequencing
- telemetry routing
- lawful store coordination
- profile and runtime discipline
- the stable C# order of operations

CradleTek does not own:

- sovereign identity custody
- membrane authority
- collapse authority
- native cognitive morphism

### SoulFrame

SoulFrame is the CME-specific mediated protected self-state constructor and membrane.

It owns:

- issuance of mediated `cSelfGEL`
- intake of collapse candidates
- legality filtering
- membrane-safe transformation
- return-path narrowing before custody routing

SoulFrame does not own:

- broad runtime orchestration
- sovereign identity authorship
- direct active cognition
- Prime publication authority

### AgentiCore

AgentiCore is the bounded personification and actualization layer for live CME cognition.

It owns:

- bounded `SelfGEL` working state
- working memory
- active cognition space
- personified runtime actualization
- candidate formation inputs for collapse

AgentiCore does not own:

- sovereign source custody
- direct Cryptic commit authority
- publication authority
- constitutional law

### Lisp-Based SLI And Engineered Cognition

Lisp-based SLI and Engineered Cognition are the native computational organism of bounded CME runtime cognition.

They own:

- symbolic restructuring
- local reflective recomposition
- decision-structure compilation
- native runtime Engrammitization
- candidate engram formation suitable for collapse mediation

They do not own:

- governance law
- custody law
- profile law
- anti-bleed law
- collapse authority

## Collapse Disposition Classes

The corrected collapse reading is staged.

Immediate Cryptic-first routing classes are first protected destinations, not final promotion outcomes:

- `route_to_cgoa`
- `route_to_cmos`

Immediate review posture is a separate axis layered over those first protected destinations:

- `no_review_required`
- `deferred_review`

Later-stage promotion or exploration classes are:

- `later_promote_to_mos`
- `eligible_for_dream_seed`
- `final_cgoa_enrichment`

The current runtime does not yet implement all of these later-stage promotion or exploration classes as first-class vocabulary.

## 1. Lawful Admission

A CME may exist only when all required preconditions are satisfied.

Admission requires:

- Sanctuary is `CONSTITUTED`
- Sanctuary is `READY`
- either the internal governance path is active or the bonded operator path is valid
- operator or governance authority scope is valid
- Mother, Father, and Steward conditions allow admission
- profile constraints are valid
- required Theater permissions are valid
- required career, job, and trade permissions are valid

A CME must be denied if any one of these conditions fails.

## 2. Instantiation

A CME begins as a working instantiation, not as the full sovereign archive of self.

The lawful instantiation path is:

`OE`, `cOE`, `MoS`, `cMoS`, `GoA`, and `cGoA` by reference
-> SoulFrame-mediated `cSelfGEL`
-> AgentiCore-bounded `SelfGEL`
-> active CME runtime

At birth, the runtime is layered as:

- Cryptic custody basis
- mediated protected self-state contour
- bounded working self-state

The sovereign archive remains below the active runtime.

## 3. Runtime Pool Propagation

### Continuity Pool

The continuity pool contains the lawful runtime basis brought into the CME at start:

- relevant `SelfGEL` slices
- relevant mediated `cSelfGEL` context
- relevant `GoA` and `cGoA` references
- operator or governance scope
- mission or task objective
- leases, profile constraints, and Theater permissions

### Active Working Pool

The active working pool is created inside AgentiCore runtime:

- working memory
- symbolic trace
- candidate decisions
- temporary reasoning structures
- task-local derived abstractions
- temporary Lisp and SLI artifacts

These belong to bounded `SelfGEL` and active cognition space, not to `cGoA` directly.

### Collapse Candidate Pool

The collapse candidate pool is prepared on close:

- candidate engram structures
- autobiographical candidate traces
- symbolic residues
- retained working-state outputs
- discardable transient noise

These are not direct custody writes. They are candidate material for mediated collapse evaluation.

## 4. Active Runtime Cognition

During active runtime, AgentiCore actualizes the CME into bounded personified working cognition.

AgentiCore may:

- carry bounded `SelfGEL`
- host working memory
- expose mission-local cognition space
- braid task context and mediated self-state into actualized runtime cognition

AgentiCore may not:

- directly mutate `cSelfGEL`
- directly write `cGoA`
- directly author `MoS` or `cMoS`
- bypass SoulFrame
- bypass governance

## 5. Native Engrammitization

Native Engrammitization is the process by which bounded runtime cognition becomes candidate engram-bearing structure suitable for lawful collapse evaluation.

This occurs through Lisp-based SLI and Engineered Cognition inside AgentiCore runtime.

Native Engrammitization may:

- restructure symbolic forms
- compile decision structure
- produce symbolic trace
- produce candidate engram structures
- produce symbolic residues
- produce autobiographical candidate traces

Native Engrammitization may not:

- define governance law
- define custody law
- define profile law
- define anti-bleed law
- directly route itself into custody destinations

Symbolic trace alone is not an engram candidate.

An engram candidate exists only when explicit native Engrammitization has produced candidate engram structure suitable for collapse evaluation.

## 6. SoulFrame-Mediated Collapse Evaluation

Collapse occurs in three stages.

### Stage A: SelfGEL Close

AgentiCore ends the active runtime and prepares:

- candidate engram structures produced through native Engrammitization
- autobiographical candidate traces
- symbolic residues
- retained working-state outputs
- discardable transient noise

### Stage B: SelfGEL To cSelfGEL Evaluation

SoulFrame receives the collapse candidate and classifies each part as:

- admissible
- transformed
- redacted
- denied
- deferred

SoulFrame remains the membrane on both issue and return.

### Stage C: cSelfGEL To Destination Routing

Only after mediated evaluation may collapse results route first into:

- `cGoA` for non-autobiographical protected contextual residue
- `cMoS` for unresolved autobiographical or SelfGEL-identified protected residue

Review state is layered on top of that first-route holding decision:

- `deferred_review` means protected material is already held in `cGoA` or `cMoS` while later governance or HITL review remains required
- `no_review_required` means the first protected hold is sufficient for the current bounded phase

Only later, after deeper curation, may material be promoted toward:

- `MoS`
- Dream-seed eligibility
- later `cGoA` enrichment payloads for non-personification improvement

The corrected model therefore distinguishes:

- first `cGoA` peel for unresolved non-autobiographical contextual residue
- first `cMoS` holding for unresolved autobiographical or SelfGEL-identified protected residue
- later `cGoA` enrichment for self-improving but non-personification material
- later `MoS` promotion only after deeper protected curation

No direct active-runtime write may bypass this mediated route.

## 7. Dream Relation

Dream is a later intake class, not the default sink for collapse residue.

Dream-eligible material must be explicitly classified as:

- suitable for Dream-seed retention
- not constitutive runtime authority
- not a substitute for Cryptic custody

Material not explicitly classified for Dream may not silently route there.

## 8. Close Conditions

A CME may be considered lawfully closed only when:

- active runtime cognition has ended
- collapse candidates have been evaluated through SoulFrame
- routing disposition has been explicitly resolved
- retained protected material has been routed lawfully
- discard and defer conditions have been recorded explicitly

## Runtime Placement Defaults

The default placement model is:

- `OE`, `cOE`, `MoS`, `cMoS`, `GoA`, and `cGoA` remain continuity, custody, and context surfaces by reference
- `cSelfGEL` is issued through SoulFrame as mediated protected self-state
- `SelfGEL` is the active bounded working self-state inside AgentiCore
- active runtime does not directly author sovereign custody

## Hard Invariants

- no direct `SelfGEL -> cGoA` write
- no direct raw-close `SelfGEL -> MoS` write
- no direct `SelfGEL -> cMoS` write
- no direct Lisp or SLI -> `cSelfGEL` mutation
- no collapse without SoulFrame mediation
- no immediate raw-close route directly to `MoS`
- no unresolved useful residue is treated as discarded merely because it was not promoted toward `MoS`
- no Dream-seed routing without explicit policy classification
- no Prime publication from active runtime residue without a later governed derivative path
- symbolic trace alone is not an engram candidate
- no collapse path may treat symbolic trace as equivalent to candidate engram structure without explicit native Engrammitization

## Planned Runtime Object Vocabulary

### `CmeAdmissionRequest`

- Purpose: request lawful CME birth under current readiness, profile, and role scope
- Producer: CradleTek orchestration and admission surfaces
- Consumer: SoulFrame and governance admission law
- Forbidden use: must not act as a direct runtime-start bypass

### `CmeAdmissionDecision`

- Purpose: record whether a CME may lawfully exist
- Producer: governance-admitted runtime law
- Consumer: CradleTek and admission orchestration
- Forbidden use: must not be treated as active runtime state by itself

### `CmeRuntimeState`

- Purpose: express current lifecycle position of the CME runtime
- Producer: runtime orchestration and lawful lifecycle evaluation
- Consumer: control plane and later lifecycle handlers
- Forbidden use: must not replace custody or governance evidence

### `CmeRuntimePlacementSnapshot`

- Purpose: describe where runtime-relevant state currently lives
- Producer: runtime lifecycle logic
- Consumer: control plane, later audits, and lifecycle checks
- Forbidden use: must not be treated as a write instruction

### `CmeContinuityPool`

- Purpose: carry the admitted continuity basis for instantiation
- Producer: custody, reference, and governance-admitted selection logic
- Consumer: SoulFrame issuance path
- Forbidden use: must not be treated as active working state

### `CmeWorkingPool`

- Purpose: carry bounded active working cognition inside AgentiCore
- Producer: AgentiCore actualization
- Consumer: active cognition and native Engrammitization
- Forbidden use: must not directly author custody destinations

### `CmeCollapseCandidate`

- Purpose: hold mediated-close candidate material
- Producer: AgentiCore close path
- Consumer: SoulFrame collapse evaluation
- Forbidden use: must not be treated as a direct save object

### `CmeCollapseEvaluation`

- Purpose: classify collapse candidate material by lawful admissibility
- Producer: SoulFrame membrane
- Consumer: routing and governance follow-through
- Forbidden use: must not silently perform routing itself

### `CmeCollapseRoutingDecision`

- Purpose: decide destination class after collapse evaluation
- Producer: governance and routing law
- Consumer: custody and deferred handling paths
- Forbidden use: must not infer missing evaluation

### `CmeCollapseDisposition`

- Purpose: classify final routing outcome for each collapse-relevant part
- Producer: collapse routing decision logic
- Consumer: custody, Dream, discard, and defer paths
- Forbidden use: must not collapse multiple destination semantics into one vague status

## Validation Scenarios

- admission success by readiness, profile, role, and Theater scope
- admission denial by readiness, profile, role, or Theater mismatch
- instantiation from custody references into mediated `cSelfGEL` and active `SelfGEL`
- active runtime cognition using bounded `SelfGEL` without custody mutation
- native Engrammitization producing candidate engram structures, not only symbolic trace
- collapse candidate mediation through SoulFrame
- routing to `cGoA`, `cMoS`, later `MoS` promotion, later Dream-seed eligibility, or defer-review
- rejection of any direct `SelfGEL -> custody` write
- rejection of any attempt to equate symbolic trace with candidate engram structure
